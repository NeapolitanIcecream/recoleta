Can AI Agents Generate Microservices? How Far are We?

Summary

本文评估AI代码代理是否能自动生成可运行的微服务，并系统比较不同上下文与提示策略的影响。结论是：代理已经能生成相对可维护的微服务代码，但正确性仍不稳定，离完全自治开发还有距离。

Problem

  • 论文解决的问题是：AI agents 能否生成满足依赖关系、API契约和系统集成要求的微服务，而不只是单个函数或文件。
  • 这很重要，因为微服务开发需要同时保证局部业务逻辑正确和跨服务接口兼容；小错误就可能导致整个系统通信失败。
  • 现有关于LLM代码生成的研究多集中在函数级任务、缺陷修复或仓库级修改，对“端到端生成微服务”这一更接近真实工程场景的问题研究不足。

Approach

  • 作者设计了一个经验性评测：共生成 144 个微服务实现，覆盖 3 个代理、4 个项目、2 种提示策略、2 种生成场景。
  • 两种场景分别是：incremental generation（在已有系统中删去目标服务，让代理补全；保留上下文和测试）与 clean state generation（只给需求，从近乎空白状态生成）。
  • 两种提示策略分别是：P1 最小提示（服务名+需求路径）和 P2 详细提示（再加上实现摘要），用来研究“更多上下文是否真的更好”。
  • 评估维度包括：功能正确性（incremental 用单元测试通过率，clean state 用依赖服务的集成测试通过率）、代码质量（SLOC、圈复杂度、认知复杂度）以及效率（token、成本、生成时间）。
  • 核心机制可以用最简单的话说：让不同AI coding agents 在不同上下文条件下“写一个完整微服务”，然后用自动化测试和代码质量指标来判断它到底写得对不对、好不好、值不值。

Results

  • 在 incremental generation 中，最小提示 P1 反而优于更详细的 P2，单元测试通过率大约在 50%–76% 之间，说明已有代码库上下文通常足够，额外摘要未必有帮助。
  • 在 clean state generation 中，集成测试通过率更高，达到 81%–98%，表明代理在只依据需求时，往往仍能较好遵守API contract并完成跨服务对接。
  • 生成代码的复杂度低于人工基线：论文明确声称AI生成代码在复杂度指标上更低，意味着代码可能更易维护；但作者也提醒，这种“低复杂度”不一定等价于更健壮，可能意味着缺少防御性处理。
  • 效率方面，不同代理差异明显，平均每个微服务生成时间约 6–16 分钟；论文还记录了 token 和成本，但摘录中未给出更细的分项数值。
  • 数据集与实验规模方面，研究覆盖 4 个正式项目（另有 1 个试点项目）、每个项目 3 个微服务任务、总计 144 次生成实验，是作者声称的首个系统性微服务生成代理评测。
  • 总体结论是：AI agents 已经可以生成“能工作的一部分”微服务，并展现出较好的API契约遵循与可维护性，但由于正确性不一致且仍需人工监督，尚不能实现完全自主的微服务生成。
