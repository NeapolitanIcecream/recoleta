Summary

本文提出一种把自然语言环境规格自动生成成可执行离散事件世界模型的方法，核心基于 DEVS 形式化与分阶段 LLM 代码生成。它还给出一种基于事件轨迹和规格约束的评测方式，用于在没有唯一真值实现时验证模型是否符合需求。

Problem

  • 现有世界模型常在两端摇摆：手工模拟器可靠但难适配，隐式神经模型灵活但难约束、难验证、长程易漂移。
  • 对于由事件顺序、时间和因果主导的环境（如队列、工作流、分布式系统、多智能体消息协调），需要一种既可在线生成又可长期一致的显式世界模型。
  • 自然语言规格通常没有唯一正确代码实现，因此不能仅靠参考实现比对；需要基于可观察行为判断是否满足规格，这对可复现评测很重要。

Approach

  • 使用 DEVS 作为世界模型表示：把环境拆成原子组件和耦合组件，每个组件显式定义状态、输入输出事件、内部/外部转移和时间推进。
  • 采用两阶段 LLM 生成流水线：先做结构生成（组件层级、端口、交互图、PlanTree），再做行为生成（各原子组件的事件逻辑与时间逻辑）。
  • 通过接口契约/JSON schema 约束生成结果，使每个组件可在局部上下文中独立生成，同时保持全局一致，支持并行生成和在线按需合成。
  • 生成的模拟器暴露标准化执行接口，运行时输出统一 schema 的 JSONL 事件轨迹，记录时间、实体、事件类型和负载。
  • 评测不依赖代码级真值，而是将轨迹对照从规格导出的时序约束与语义不变量进行验证，并在违例时提供定位诊断。

Results

  • 摘要与节选中没有提供定量实验结果，未给出具体数据集、通过率、准确率、开销或相对基线提升数字。
  • 论文的最强具体主张是：所生成的离散事件世界模型可在长时域 rollout 中保持更一致的行为，并且比隐式世界模型更容易约束、验证和调试。
  • 作者宣称该方法支持从自然语言按需高效合成可执行模拟器，尤其适用于队列/服务系统、工作流、具身任务规划、消息驱动多智能体协调等离散事件环境。
  • 作者还声称轨迹级验证可实现可复现的黑盒评测与局部化错误诊断，即便在不存在唯一参考实现时也能检查规格符合性。
