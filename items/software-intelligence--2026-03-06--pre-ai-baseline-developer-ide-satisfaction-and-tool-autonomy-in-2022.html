Summary

这篇论文提供了生成式 AI 普及前开发者 IDE 满意度的量化基线，基于 2022 年 7 月的 1,155 名开发者调查。核心发现是：当时 IDE 总体满意度已很高，而“能否自由选择工具”是最强的满意度预测因素。

Problem

  • 论文要解决的问题是：在 AI 编程工具大规模进入开发流程之前，开发者对 IDE、工具选择自由度、云 IDE 的真实态度到底是什么；这很重要，因为没有前 AI 基线，就无法判断后续 AI 工具到底改变了什么。
  • 仅看 AI 时代的生产率或满意度研究会混淆因果：不知道变化来自 AI，还是原本工具生态、工作流、开发者经验差异。
  • 另一个关键问题是，表面高满意度是否掩盖了工具流失、云 IDE 低采用、以及组织限制带来的潜在不满。

Approach

  • 作者在 2022 年 7 月做了一次横截面在线调查，收集 1,173 份完整回复，其中 1,155 份用于满意度分析，覆盖 52 个国家开发者。
  • 用 0–10 推荐评分衡量 IDE 满意度，并同时分析 工具选择自主性、经验水平、实验型用户倾向、云 IDE 采用/障碍等变量。
  • 统计上结合了均值/置信区间、t 检验、ANOVA、线性回归，以及有序 logistic 回归稳健性检验；核心目的是找出满意度的最强解释因素。
  • 方法上最简单的理解是：先测大家对当前 IDE 有多满意，再比较“可自由选工具的人”和“受限制的人”、不同经验层级的人，以及云 IDE 用户之间的差异。

Results

  • 总体 IDE 满意度很高：平均分 8.14/10，95% CI [8.01, 8.25]；NPS 为 34.7，其中 50.8% 是 promoter，33.1% passive，16.1% detractor。
  • 工具生态高度集中：Visual Studio Code 使用率 79.0%，IntelliJ IDEA 30.1%，Visual Studio 26.3%。
  • 自主性是最强预测因素：可自由选 IDE 的开发者满意度 8.44，受限者 7.73，差值 0.71 分；t(1153) = 6.18，p < 0.001，Cohen's d = 0.49。多变量回归中，自主性系数 β = 0.51 [0.24, 0.78], p < 0.001；有序 logistic 稳健性分析给出 OR = 1.61 [1.30, 1.99]。
  • 经验与满意度正相关：ANOVA F(4,1151)=12.51, p<0.001, η²=0.040；新手（<1 年）均值 6.53，20+ 年开发者 8.41，差值 1.88 分，Cohen's d = 0.43。
  • “实验者”群体占 29.9%，虽然更爱换工具、使用技术更多（5.9 vs 5.2，t = 5.38, p < 0.001），但满意度并无显著差异（t = 0.43, p = 0.67）。
  • 云 IDE 常规采用极低，仅 4.3% 为 regular users，16.0% occasional users，34.2% 计划采用，44.6% 无计划；摘要还指出 40.1% 受访者将网络依赖视为主要障碍。作者还声称 IDE 留存差异显著：VS Code 68.5%，传统 IDE 仅 3.9%–25%。这些结果共同构成后 AI 时代研究“生产率—满意度错位”的前基线。
