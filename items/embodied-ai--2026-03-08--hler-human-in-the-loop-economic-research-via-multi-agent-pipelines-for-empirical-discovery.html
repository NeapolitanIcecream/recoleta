Summary

HLER 是一个面向经济学实证研究的人类在环多代理流水线，用数据约束的选题与迭代审稿来减少 LLM 在科研自动化中的空想与失真。它强调不是完全自治，而是在关键科学决策点保留人工把关。

Problem

  • 现有“AI 科学家”式系统偏向全自动，但经济学实证研究要求问题必须匹配真实数据、识别策略要合理、结论还要经过人类判断其经济意义。
  • 普通 LLM 容易提出不可行或幻觉式假设，例如数据里根本没有相关变量，或数据结构不支持所需设计。
  • 这很重要，因为实证社科研究的可信度依赖数据审计、可复现分析和反复修订，而不是只生成一篇看起来像论文的文本。

Approach

  • HLER 将流程拆成多个专职 agent：数据审计、数据画像、问题生成、数据收集、计量分析、论文撰写、自动审稿，由一个 orchestrator 串联。
  • 核心机制是dataset-aware hypothesis generation：先读取数据表结构、变量可用性、缺失模式、分布和相关性，再让 LLM 生成研究问题，从源头约束不切实际的选题。
  • 系统包含两个闭环：问题质量环（生成 → 可行性筛选 → 人类选择）和研究修订环（审稿 → 追加分析/稳健性检验 → 改稿）。
  • 在关键节点保留人工决策门：人类研究者负责选题和是否最终发布，其余繁琐步骤尽量自动化。
  • 计量部分主要由程序化统计库执行，如 OLS、固定效应、DID、event-study；LLM 主要负责推理、规划和写作。

Results

  • 在 3 个数据集、14 次完整 pipeline 运行中，数据感知选题生成了 79 个候选问题，其中 69 个可行（87%）；无约束生成是 82 个中 34 个可行（41%），即不可行比例从 59% 降到 13%。
  • 无约束生成的主要失败原因包括：变量不存在占失败的 42%，设计与数据结构不兼容占 35%。
  • 端到端完成率为 12/14 = 86%；失败的 2 次都卡在固定效应估计的稀疏子样本收敛问题上，但系统能记录错误并优雅停止。
  • 修订环中，12 个完成运行的 reviewer 平均总分从初稿 4.8 ± 0.9 提升到一轮修订后 5.9 ± 0.7，最终稿达到 6.3 ± 0.6；最大提升来自 清晰度 +2.1 分 和 识别可信度 +1.4 分，而新颖性仅 +0.3 分。
  • 典型 reviewer 请求包括：额外稳健性检验（10/12）、更清楚的识别策略讨论（8/12）、更好的变量说明（7/12）；多数运行在 2–3 轮内收敛，文中也称通常 2–4 轮。
  • 单次运行大约需要 20–25 分钟，平均 API 成本 $0.8–$1.5/次；作者对比 AI Scientist 报告的 $6–$15/篇，声称 HLER 成本更低。案例研究中，CHNS 任务从 5,563 词初稿修到 7,282 词终稿，review 分数从 4.6 升到 6.5，识别可信度从 3.2 升到 5.8。
