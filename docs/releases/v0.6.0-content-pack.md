# Recoleta v0.6.0 Content Pack

Use this file for a demo, screenshots, or a longer release post.

## Demo outline

### 1. Show the default Huldra path

Start a Huldra daemon, then use an enabled arXiv source without selecting the
old direct mode:

```yaml
sources:
  arxiv:
    enabled: true
    mode: pool

arxiv_pool:
  enabled: true
  backend: huldra
  huldra_base_url: http://127.0.0.1:8765
```

Show Huldra `/healthz` and `/v1/status`. Explain that metadata requests,
request budgets, queue state, and cooldown now belong to Huldra.

### 2. Prove dry-run isolation

Run a Fleet day plan:

```bash
uv run recoleta fleet run day --manifest ./fleet/fleet.yaml --dry-run --json
```

Point out `status: skipped` and `reason: dry_run` for pre-sync and readiness.
The Huldra request and queue counters should not change.

### 3. Configure bounded Fleet scheduling

Add schedules only to the top-level Fleet manifest:

```yaml
daemon:
  schedules:
    - workflow: day
      interval_minutes: 60
      catch_up_windows: 2
    - workflow: week
      weekday: mon
      hour_utc: 2
      minute_utc: 0
      catch_up_windows: 2
```

```bash
uv run recoleta fleet daemon start --manifest ./fleet/fleet.yaml
```

Explain that mature windows are closed UTC periods, missing windows are chosen
oldest first, and one host-local lease covers all child runs.

### 4. Inspect rate-limit behavior

Use Huldra status and Recoleta diagnostics to show upstream request count, 429
count, cooldown, queue depth, oldest pending request, and worker errors. A
content 429 must not produce immediate retries or an HTML-to-PDF fallback.

## Longer launch post

Recoleta v0.6.0 consolidates arXiv metadata access in Huldra. Standard installs
include the reviewed Huldra 0.4.0 revision, enabled arXiv sources default to its
pool backend, and direct or local SQLite operation requires an explicit
deprecated rollback setting. Recoleta does not silently bypass an unavailable
broker.

Fleet automation now works on closed UTC periods and reconciles a bounded
number of oldest gaps. One top-level scheduler coordinates every child in the
manifest, while a host-local lease prevents manual and scheduled sequences from
interleaving on the same machine. Bounded pre-sync asks Huldra to prepare mature
metadata windows before workflows begin.

Content enrichment uses cross-process admission for arxiv.org HTML, PDF, and
e-print requests. Shared cooldown contains 429 responses without retry or
fallback fanout. Dry-runs avoid Huldra entirely, so operators can inspect a plan
without queueing metadata work.
