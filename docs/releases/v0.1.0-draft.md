# Recoleta v0.1.0 Draft Release Notes

Recoleta turns noisy research sources into a local-first AI research radar.

## Highlights

- Multi-source ingest across arXiv, Hacker News RSS, Hugging Face Daily Papers,
  OpenReview, and RSS feeds
- Structured LLM analysis with topic scoring, filtering, and topic streams
- Daily trend briefs and follow-on idea briefs from stored local state
- Markdown-first publishing with optional Obsidian, Telegram, PDFs, and a
  public static site
- Docker runtime images, Compose workflow, healthcheck, and JSON stats
- starter presets plus a first-output tour with sample screenshots

## Good first paths

- Run locally with the 5-minute quickstart in `README.md`
- Follow `docs/guides/first-output-tour.md` to see what success should look like
- Start from a preset in `presets/`
- Build and publish the site with `uv run recoleta site build` or
  `uv run recoleta site gh-deploy`
- Configure the repo surface with `docs/releases/github-launch-handoff.md`
- Reuse channel copy from `docs/releases/v0.1.0-launch-kit.md`

## Suggested release title

`v0.1.0: local-first AI research radar baseline`

## Suggested release blurb

Recoleta is now packaged as a public baseline release for running a local-first
AI research radar. This release centers on three surfaces: multi-source ingest,
publishable trend and idea briefs, and a GitHub Pages-ready static site. It
also includes starter presets and a first-output tour so new users can get from
clone to recognizable output faster.
