# Recoleta v0.1.0 Release Notes

Recoleta now has a public baseline release for running a local-first AI
research radar.

## What is in this release

- Multi-source ingest across arXiv, Hacker News RSS, Hugging Face Daily Papers,
  OpenReview, and RSS feeds.
- Structured LLM analysis with topic scoring and filtering.
- Daily trend briefs and follow-on idea briefs built from stored local state.
- Markdown-first publishing with optional Obsidian, Telegram, PDFs, and a
  public static site.
- Docker images, a Compose workflow, healthchecks, and JSON workspace stats.
- Starter presets plus a first-output tour with screenshots.

## Good places to start

- Follow the 5-minute quickstart in [`README.md`](../../README.md).
- Use [`docs/guides/first-output-tour.md`](../guides/first-output-tour.md) to
  confirm what a successful first run should create.
- Pick a starter config from [`presets/`](../../presets/).
- Build the site with `uv run recoleta run site build` or publish it with
  `uv run recoleta run deploy`.
- Reuse launch copy from
  [`docs/releases/v0.1.0-launch-kit.md`](./v0.1.0-launch-kit.md).

## Suggested release title

`v0.1.0: local-first AI research radar baseline`

## Suggested release summary

Recoleta v0.1.0 packages the current project as a public starting point for
tracking AI research from one local workspace. It pulls from papers, discussion
threads, and feeds, then turns that stream into Markdown notes, trend briefs,
idea briefs, and a static site. This release also adds starter presets and a
first-output tour so new users can get to recognizable output faster.

## Validation commands

uv run ruff check .
