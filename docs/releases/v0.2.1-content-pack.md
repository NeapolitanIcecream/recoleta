## 60-second demo script

### Beat 1: the operational gap

Recoleta already produced canonical trend markdown, sibling presentation
sidecars, and a publishable static site. The remaining gap was the last-mile
operator workflow for sending trend email batches without maintaining a
separate email content path.

### Beat 2: what v0.2.1 changes

v0.2.1 adds `recoleta run email preview|send` and the matching
`recoleta fleet run email preview|send` commands. Operators configure an
ordered `EMAIL.granularities` set, optionally filter it with repeatable
`--granularity`, and render email batches directly from stored canonical
outputs.

### Beat 3: link resolution and delivery safety

Email rendering uses a private site email link-map artifact so internal links
point at the already deployed public site instead of local filesystem paths.
Send runs also require a publicly reachable primary trend page and reject mixed
partially sent recipient batches unless the operator passes `--force-batch`.

### Beat 4: delivery model

Manual trend email is configured separately from `PUBLISH_TARGETS`. Delivery
uses `EMAIL` config plus env-only `RECOLETA_RESEND_API_KEY`, and persists
dedupe-oriented state in `trend_deliveries`.

### Beat 5: where to start

Start with the README for setup, the usage recipes for preview/send commands,
and the configuration doc for the exact `EMAIL` fields and environment
requirements.

## Case-study angles

### 1. One weekly or monthly operator email

- Hook: `Turn the latest weekly and monthly trend notes into site-linked email batches.`
- Show:
  - `recoleta run site build`
  - `recoleta run email preview --granularity week --granularity month`
  - `recoleta run email send --granularity week --granularity month`
- Main claim:
  `v0.2.1 adds the last-mile email step without changing the canonical content path.`

### 2. Fleet maintainer sending one child instance

- Hook: `Send a manual trend email for one child instance without flattening the fleet into one shared runtime.`
- Show:
  - `recoleta fleet site build --manifest ...`
  - `recoleta fleet run email preview --manifest ... --instance agents-radar`
  - `recoleta fleet run email send --manifest ... --instance agents-radar`
- Main claim:
  `Fleet email workflows stay instance-scoped and explicit.`

### 3. Safe resend boundary

- Hook: `Avoid ambiguous partial resend state when a batch only reached some recipients.`
- Show:
  - normal `recoleta run email send`
  - mixed-batch failure
  - deliberate `recoleta run email send --granularity week --force-batch`
- Main claim:
  `Batch-oriented safety checks make resend intent explicit instead of implicit.`
