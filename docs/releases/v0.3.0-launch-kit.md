## GitHub release body

Recoleta v0.3.0 adds Huldra-backed arXiv pool support for workspaces that want
arXiv metadata ready before instance ingest, analysis, publishing, and trend
generation run. Configure `sources.arxiv.mode: pool`,
`arxiv_pool.backend: huldra`, and install with `uv sync --extra huldra` to use
the Huldra path.

The release also adds `recoleta arxiv-pool sync|backfill|worker`,
`recoleta inspect arxiv-pool freshness`, and `recoleta admin arxiv-pool gc`.
Single-instance and fleet workflows now evaluate pool readiness with maturity
lag and `off|warn|strict` gates, so strict runs can stop before incomplete
arXiv windows produce partial downstream output.

Starter presets that include arXiv now use Huldra-backed pool mode by default.
This release also adds `recoleta inspect localization`, batch-first manual
trend email runs across configured granularities, and workflow timing metrics
with fleet day benchmark helpers.

## Short launch post

Recoleta v0.3.0 is out. This release adds Huldra-backed arXiv pool mode, pool
freshness inspection, and readiness gates for single-instance and fleet
workflows. Strict runs can now stop before missing or immature arXiv windows
flow into analysis, publishing, trends, translation, or site builds.

Starter presets that use arXiv now default to Huldra-backed pool mode. The same
release also adds localization coverage inspection, batch-first manual trend
email runs across configured granularities, and workflow timing metrics with
fleet day benchmark helpers.

## Direct outreach blurb

Recoleta v0.3.0 is aimed at arXiv-heavy operators. It lets Recoleta read arXiv
metadata from Huldra or a local SQLite pool, prewarm the windows a workflow
needs, inspect freshness, and use readiness gates to avoid publishing from
partial arXiv coverage.
