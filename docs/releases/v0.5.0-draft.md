# Recoleta v0.5.0 Release Notes

Recoleta v0.5.0 makes generated Trends and Ideas easier to audit. Generation
now records exact evidence reads, requires independent sources, suppresses
low-evidence windows, and removes stale downstream projections.

The release also adds stage-specific LLM configuration, static collection
pagination, web mathematics, and a revised site and email reading experience.

## What is in this release

- Set `ANALYZE_LLM_MODEL`, `TRENDS_LLM_MODEL`, `IDEAS_LLM_MODEL`, and
  `TRANSLATION_LLM_MODEL` independently, or use `--model` for a workflow or
  stage override. Model provenance controls one-time refreshes and reuse.
- Trend and Idea agents must read the exact evidence chunks they cite and meet
  independent-source quality gates. Low-evidence windows are stored as
  suppressed instead of being expanded into weak prose.
- JSON output keeps `status: ok` for a completed command and adds
  `artifact_status` to distinguish succeeded and suppressed artifacts.
- Analyze budget receipts prevent a bounded day workflow from repeating an
  already completed analysis budget under the same effective settings.
- Trends, Ideas, Topics, Archive, and topic detail pages paginate at build time.
  Existing first-page and detail URLs remain valid.
- Inline and display formulas render as selectable MathML on web pages. Email
  keeps formulas as readable TeX for broad client compatibility.
- The site and email presentation now use localized interface copy, semantic
  item metadata, deduplicated sources, accessible markup, and a plain-text
  email alternative.
- Translation scope handling, Outlook CTA alignment, batched metrics, logging,
  and artifact-retention safety have targeted fixes.

## Compatibility

No database migration or required configuration change is needed. The new
stage-specific model settings are optional and fall back to the global model.
Selecting a different effective model can trigger a one-time refresh of
artifacts whose stored provenance differs; previously published items are not
delivered again.

Strict evidence gates intentionally suppress sparse Trend or Idea windows. A
suppressed Trend invalidates Ideas derived from it and cleans stale generated
files after the durable state is recorded.

## Suggested release title

`v0.5.0: Evidence-gated artifacts and stage-specific models`

## Suggested release summary

Recoleta v0.5.0 adds exact evidence-read traces and independent-source gates to
Trend and Idea generation, suppresses weak windows, and cleans stale derived
artifacts. It also adds stage-specific LLM settings, provenance-aware refreshes,
static collection pagination, MathML web rendering, and revised site and email
presentation. Translation scope, Outlook markup, metric growth, dependency
logging, and artifact garbage collection receive targeted fixes.
