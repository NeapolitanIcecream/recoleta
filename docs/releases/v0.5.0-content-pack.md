# Recoleta v0.5.0 Content Pack

Use this file for a demo, screenshots, or a longer release post.

## Demo outline

### 1. Configure models by stage

Show the optional settings and a one-run override:

```yaml
analyze_llm_model: openai/gpt-5-mini
trends_llm_model: anthropic/claude-sonnet-4-5
ideas_llm_model: anthropic/claude-sonnet-4-5
translation_llm_model: openai/gpt-5-mini
```

```bash
uv run recoleta run week --date 2026-07-13 --model openai/gpt-5.1
```

Explain that precedence is explicit override, stage setting, then global model.
Stored provenance determines whether existing work can be reused.

### 2. Show succeeded and suppressed artifacts

Run a Trend command with JSON output:

```bash
uv run recoleta stage trends --granularity week --date 2026-07-13 --json
```

Point out `artifact_status`. A command can finish successfully while reporting
that a sparse window was suppressed by the evidence gate. Derived Ideas and
stale projections follow that canonical status.

### 3. Browse a large generated site

Build or serve a site containing more than one page of Trends, Ideas, Topics,
or Archive entries:

```bash
uv run recoleta run site build
uv run recoleta run site serve
```

Show that page one keeps its original URL and later pages use paths such as
`/trends/page/2/`. Check desktop and mobile pagination controls.

### 4. Compare web and email mathematics

Open a generated brief containing inline and display TeX. The web page renders
selectable MathML, while HTML and plain-text email keep readable TeX so one
payload works across Gmail and Outlook without remote equation images.

## Longer launch post

Recoleta v0.5.0 changes how generated research artifacts are accepted and
reused. Trend and Idea agents record the exact document chunks they inspect and
must meet independent-source gates. Sparse windows are stored as suppressed;
they do not produce padded prose or leave current-looking files downstream.

Operators can choose separate models for analysis, trend synthesis, idea
generation, and translation. Workflow and stage `--model` overrides take
priority, and stored provenance determines whether an artifact is reused or
refreshed. Previously published items keep their delivery state during model
refreshes.

The publishing surface now paginates large collections at build time and
renders inline and display mathematics as semantic MathML. Site and email
layouts use localized interface copy, clearer item metadata, deduplicated
sources, accessible markup, and plain-text email output. Targeted fixes bound
metric growth and make translation reruns, Outlook buttons, logs, and artifact
garbage collection safer.
