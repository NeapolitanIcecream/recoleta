## GitHub release body

Recoleta v0.6.0 makes Huldra 0.4.0 the default and standard dependency for
arXiv metadata. Enabled arXiv sources share one broker for caching, queueing,
request budgets, and durable cooldown. The deprecated direct adapter and local
SQLite pool remain available only when selected explicitly; there is no
automatic fallback from Huldra.

Fleet operators can declare schedules once in the top-level manifest. Day,
week, and month jobs process closed UTC periods, reconcile a bounded number of
oldest gaps, and hold one host-local lease across the complete child sequence.
Bounded Huldra pre-sync can fill mature metadata windows before child workflows
start.

Recoleta processes now coordinate arxiv.org HTML, PDF, and e-print admission.
A 429 creates shared cooldown instead of immediate retry or fallback fanout.
Dry-runs skip Huldra pre-sync and readiness calls, preserving their
zero-side-effect contract.

The release pipeline now installs the built wheel with current unlocked
dependencies and runs the installed CLI, preventing lock-only validation from
masking dependency API breaks.

Back up the Huldra database before its first 0.4.0 startup because Huldra
advances the store to schema 8. Recoleta adds no database migration in this
release. See the release notes for rollback settings and scheduler constraints.

metadata, with no automatic fallback to the deprecated direct adapter or local
