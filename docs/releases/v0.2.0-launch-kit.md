## GitHub release body

Recoleta v0.2.0 makes the project easier to run as an everyday workflow. The
CLI now centers on `run`, `fleet`, `inspect`, and `repair`, while
multi-instance deployments move to child configs plus a fleet manifest instead
of shared `topic_streams` state.

This release also adds aggregate run inspection, broader `--json` support,
local fleet site preview, and sidecar-first localized trend and idea outputs
so publishing and auditing are easier to automate.

## Short launch post

Recoleta v0.2.0 is out. It keeps the local-first AI research radar model from
the first release, but now the main CLI is workflow-first, fleets are
instance-first, and inspect or repair flows are easier to script. The same
workspace still turns papers, discussion threads, and feeds into Markdown
notes, trend briefs, idea briefs, and a publishable site.
