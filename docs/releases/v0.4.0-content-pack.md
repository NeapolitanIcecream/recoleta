## Demo outline

### 1. Inspect a workflow replay plan

Show a dry-run plan before running a historical workflow:

```bash
uv run recoleta run week --date 2026-03-02 --dry-run --json
uv run recoleta fleet run week --manifest ./fleet/fleet.yaml --date 2026-03-02 --dry-run --json
```

Suggested line:

`v0.4.0 lets operators see the planned steps, skip reasons, and expensive-step count before a replay creates run rows or spends provider calls.`

### 2. Replay the missing work

Show the same workflow without `--dry-run`:

```bash
uv run recoleta run week --date 2026-03-02
uv run recoleta fleet run week --manifest ./fleet/fleet.yaml --date 2026-03-02
```

Suggested line:

`Week and month workflows still ensure lower-level windows, but completed daily or weekly trend and idea task sets are skipped by default.`

### 3. Regenerate only when requested

Show the intentional regeneration path:

```bash
uv run recoleta run week --date 2026-03-02 --force
```

Suggested line:

`Use --force when the goal is regeneration rather than backfilling missing work.`

### 4. Check email output after a deploy

Show the manual email preview path after a fleet deploy:

```bash
uv run recoleta fleet run deploy --manifest ./fleet/fleet.yaml
uv run recoleta run email preview --granularity week
```

Suggested line:

`The email-ready site is refreshed during fleet deploy, email buttons render more reliably in Outlook, and truncated summary excerpts are marked in the email body.`

## Longer launch post

Recoleta v0.4.0 focuses on replay safety. Operators often need to rerun a day,
week, or month after a partial run, a provider failure, or a fleet migration.
Before this release, parent-window replays could regenerate lower-level work
that was already present.

The new workflow planner turns `run day|week|month` and
`fleet run day|week|month` into ensure/backfill commands. They inspect existing
state, run missing work, and skip completed expensive generation by default.
`--dry-run --json` returns the plan before the command writes run rows, emits
outputs, or spends provider calls.

For week and month workflows, recursive lower-level generation is now a
first-run task-set gate. If daily or weekly trend and idea outputs already
exist inside the parent window, Recoleta skips that lower-granularity task set.
Use `--force` when you intentionally want a full regeneration.

The same release also cleans up several operator-facing edges. Pydantic AI RAG,
trend, and idea agents now use `RECOLETA_LLM_API_KEY` and
`RECOLETA_LLM_BASE_URL` consistently with LiteLLM-backed stages. The `huldra`
extra installs the pinned `huldra-arxiv` package. Fleet deploy refreshes the
email-ready site before manual trend email delivery, Outlook email buttons are
more reliable, email summaries mark truncation, and lower-granularity trend
backfill fills only untouched outputs.
