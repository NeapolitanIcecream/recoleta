## GitHub release body

Recoleta v0.4.0 makes day, week, and month workflow replays safer to run. The
workflow planner now inspects existing state before execution, runs missing
work, and skips completed expensive generation by default.

Use `--dry-run --json` on `recoleta run day|week|month` or
`recoleta fleet run day|week|month` to see the exact plan, skip reasons, and
planned expensive-step count before a replay creates run rows or spends
provider calls. Use `--force` when you intentionally want to regenerate
expensive content.

This release also aligns Pydantic AI RAG, trend, and idea agents with
`RECOLETA_LLM_API_KEY` and `RECOLETA_LLM_BASE_URL`, switches the `huldra` extra
to the pinned `huldra-arxiv` package, refreshes the email-ready site during
fleet deploy, improves Outlook email buttons, marks truncated trend email
summaries, and keeps lower-granularity trend backfills from regenerating
existing outputs.

## Short launch post

Recoleta v0.4.0 is out. Day, week, and month workflows now use an
ensure/backfill planner that inspects existing state, runs missing work, and
skips completed expensive generation by default.

Run `--dry-run --json` before a replay to see the exact plan, skip reasons, and
planned expensive-step count. The same release also fixes fleet deploy email
readiness, Outlook email buttons, trend email truncation markers, lower-level
trend backfills, Huldra optional dependency wiring, and Pydantic AI LLM routing.

## Direct outreach blurb

Recoleta v0.4.0 is aimed at operators who replay historical or fleet workflows.
It adds an ensure/backfill planner for day, week, and month commands, so reruns
can skip completed expensive generation unless you explicitly use `--force`.
