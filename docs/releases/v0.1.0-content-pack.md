# Recoleta v0.1.0 Content Pack

Use this file when you need more than a short release blurb: demo narration,
case-study framing, or longer-form launch writing.

## 60-second demo script

### Beat 1: problem

`If you follow AI research seriously, your inputs are scattered across papers, Hacker News, OpenReview, newsletters, and RSS. The problem is not finding more sources. The problem is turning them into something you can actually review.`

### Beat 2: setup

`Recoleta is a local-first AI research radar. It pulls those sources into one workspace, keeps state in SQLite, and turns the stream into publishable outputs instead of another unread backlog.`

### Beat 3: first result

`After a first run, you get Markdown inbox notes and a latest index. Once you add trends and ideas, the same workspace turns into trend briefs, idea briefs, and a static site.`

### Beat 4: why this is different

`The point is not just aggregation. Recoleta keeps the pipeline local, preserves source links, and lets you publish to notes, chat, and the web from the same underlying state.`

### Beat 5: close

`If you want a repeatable way to track AI research without living in tabs, start with the preset gallery or the first-output tour.`

## Case-study angles

### 1. Agents radar

Use when the audience is AI builders, code-agent users, or infra teams.

- Hook: `Track agent tooling without mixing product noise, papers, and forum chatter by hand.`
- Setup: start from `presets/agents-radar.yaml`
- Show:
  - first run to `latest.md` and `Inbox/`
  - one trend brief
  - one idea brief
  - stream page on the static site
- Main claim: `Recoleta turns scattered agent-system updates into a brief you can review and publish from one workspace.`

### 2. Robotics radar

Use when the audience is embodied-AI or robotics researchers.

- Hook: `Keep up with robotics and VLA work without building a custom paper-watching stack first.`
- Setup: start from `presets/robotics-radar.yaml`
- Show:
  - arXiv-heavy ingest flow
  - trend brief for a robotics-heavy window
  - site stream page
- Main claim: `Recoleta gives robotics teams a repeatable way to move from paper feeds to shared trend notes.`

### 3. arXiv digest

Use when the audience wants the simplest first-run story.

- Hook: `Start with papers only, then add more sources later if the baseline is useful.`
- Setup: start from `presets/arxiv-digest.yaml`
- Show:
  - minimal config
  - first run output
  - one trend brief
- Main claim: `Recoleta can start as a simple paper-first digest instead of forcing a full multi-source setup on day one.`
