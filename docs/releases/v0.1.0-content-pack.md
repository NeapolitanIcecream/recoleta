# Recoleta v0.1.0 Content Pack

Use this file when you need more than a short release blurb: demo narration,
case-study framing, or a longer launch post.

## 60-second demo script

### Beat 1: the input problem

`If you follow AI research closely, your inputs probably come from papers, discussion threads, newsletters, and RSS feeds. Recoleta pulls those sources into one workspace so you can review them together instead of tracking them by hand.`

### Beat 2: what Recoleta does

`Recoleta is local-first. It stores state in SQLite, writes Markdown first, and keeps site pages, PDFs, and chat delivery as derived outputs.`

### Beat 3: what the first run gives you

`After the first run, you get Markdown inbox notes and a latest index. Once you add trends and ideas, the same workspace also gives you trend briefs, idea briefs, and a static site.`

### Beat 4: why the workflow matters

`Recoleta keeps source links, preserves local state, and lets you publish from the same underlying corpus instead of juggling separate tools for collection and publishing.`

### Beat 5: where to start

`If you want to try it quickly, start with the preset gallery or the first-output tour.`

## Case-study angles

### 1. Agents radar

Use when the audience is AI builders, code-agent users, or infra teams.

- Hook: `Track agent tooling without manually mixing papers, launch posts, and forum threads.`
- Setup: start from `presets/agents-radar.yaml`
- Show:
  - first run to `latest.md` and `Inbox/`
  - one trend brief
  - one idea brief
  - the matching stream page on the static site
- Main claim:
  `Recoleta turns scattered agent-system updates into a reviewable brief from one workspace.`
