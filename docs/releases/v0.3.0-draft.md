# Recoleta v0.3.0 Release Notes

Recoleta v0.3.0 adds
[Huldra](https://github.com/NeapolitanIcecream/huldra)-backed arXiv pool
support and readiness gates for operators who want arXiv metadata to be
prewarmed before analysis, publishing, and trend generation run.

## What is in this release

- Optional Huldra-backed arXiv pool mode via `uv sync --extra huldra`,
  `sources.arxiv.mode: pool`, and `arxiv_pool.backend: huldra`.
- arXiv pool commands for local and Huldra-backed workflows:
  `recoleta arxiv-pool sync|backfill|worker`,
  `recoleta inspect arxiv-pool freshness`, and
  `recoleta admin arxiv-pool gc`.
- Single-instance and fleet readiness gates that check requested pool windows,
  apply a maturity lag, and block strict workflows before they analyze or
  publish incomplete arXiv windows.
- Fleet pre-sync support for pool-backed arXiv windows before day, week, month,
  and deploy workflows continue.
- Starter presets now use Huldra-backed arXiv pool mode by default, with
  matching examples in `README.md`, `presets/README.md`, and
  `recoleta.example.yaml`.
- `recoleta inspect localization` for local checks of translated item, trend,
  idea, materialized, and site coverage.
- Batch-first manual trend email runs for multiple configured granularities.
- Workflow timing metrics plus benchmark and comparison helpers for fleet day
  runs.

## Suggested release title

`v0.3.0: Huldra-backed arXiv pool and readiness gates`

## Suggested release summary

Recoleta v0.3.0 moves arXiv-heavy operation toward a pool-first workflow.
Workspaces can read arXiv metadata from Huldra or from Recoleta's local SQLite
pool, prewarm requested windows, inspect freshness, and let strict workflows
stop before analysis or publishing when a pool window is missing or immature.
The release also refreshes starter presets for Huldra-backed arXiv use, adds a
localization coverage inspector, and rounds out batch-oriented trend email
runs.
