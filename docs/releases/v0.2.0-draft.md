# Recoleta v0.2.0 Release Notes

Recoleta v0.2.0 turns the initial public baseline into a workflow-first release
for both single-instance and fleet-style operation.

## What is in this release

- A workflow-first CLI under `run`, `fleet`, `daemon`, `inspect`, `repair`,
  `stage`, and `admin`.
- An instance-first multi-instance model built around child configs plus a
  fleet manifest.
- Aggregate fleet workflows for `fleet run day|week|month|deploy`,
  `fleet site build`, and `fleet site serve`.
- Read-only inspection workflows for LLM config, empty-corpus diagnosis, and
  run history, plus broader `--json` coverage for scripting.
- Sidecar-first trend and idea presentation surfaces, including localized trend
  and idea sidecars for multilingual outputs.
- Structured run and artifact metadata that make failure review and replay
  safer.

## Suggested release title

`v0.2.0: workflow-first runtime and instance-first fleet`

## Suggested release summary

Recoleta v0.2.0 keeps the local-first research radar model from `v0.1.0`, but
changes the operating surface in three practical ways: the main CLI is now
workflow-first, multi-instance deployments are instance-first, and inspection
or repair flows are easier to script and audit. The same release also hardens
localized outputs by preferring sibling presentation sidecars for trend and
idea rendering.
