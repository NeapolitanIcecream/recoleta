# Recoleta v0.4.0 Release Notes

Recoleta v0.4.0 makes day, week, and month workflow replays safer to run.
Workflow commands now plan ensure/backfill work before execution, skip completed
expensive generation by default, and expose dry-run JSON with the exact plan.

The release also aligns Pydantic AI model routing with the Recoleta LLM
environment variables and includes email, fleet deploy, Huldra dependency, and
trend backfill fixes.

## What is in this release

- Idempotent ensure/backfill planning for `recoleta run day|week|month` and
  `recoleta fleet run day|week|month`.
- `--dry-run --json` plan output for day, week, and month workflows, including
  step decisions, skip reasons, and planned expensive-step counts.
- Week and month workflows now skip recursive lower-granularity trend and idea
  generation when that lower-granularity task set already exists in the parent
  window. Use `--force` when you want to regenerate it.
- Pydantic AI RAG, trend, and idea agents now use `RECOLETA_LLM_API_KEY` and
  `RECOLETA_LLM_BASE_URL` consistently with LiteLLM-backed stages.
- The `huldra` optional extra now installs the `huldra-arxiv` package from the
  pinned Huldra repository revision.
- Fleet deploy refreshes the email-ready site before manual trend email
  delivery.
- Manual trend email calls to action render more reliably in Outlook, and
  summary excerpts now show when text was truncated.
- Lower-granularity trend backfill fills only untouched lower-level outputs
  instead of regenerating existing daily or weekly trend work during a parent
  window replay.

## Suggested release title

`v0.4.0: Idempotent workflow backfills and safer email runs`

## Suggested release summary

Recoleta v0.4.0 makes workflow replays more predictable for operators. Day,
week, and month commands now plan the work they need, skip completed expensive
generation by default, and show the exact plan through dry-run JSON before a
replay spends provider calls. The release also fixes fleet deploy email
readiness, Outlook email buttons, trend email truncation markers, lower-level
trend backfills, Huldra optional dependency wiring, and Pydantic AI LLM routing.
