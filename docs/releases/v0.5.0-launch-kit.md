## GitHub release body

Recoleta v0.5.0 makes generated Trends and Ideas easier to audit. Agents now
record exact evidence reads, must meet independent-source quality gates, and
suppress low-evidence windows instead of expanding them into weak prose. A
suppressed Trend also invalidates derived Ideas and removes stale projections.

Analyze, Trends, Ideas, and Translation can now use separate LLM settings, with
`--model` as the highest-priority workflow or stage override. Stored model
provenance controls reuse and one-time refreshes without redelivering previously
published items. Analyze budget receipts also prevent completed bounded day
workflows from repeating the same analysis budget.

Publishing now includes static pagination for generated collections, selectable
MathML on web pages, readable TeX in email, localized interface copy, semantic
item metadata, accessible email markup, and a plain-text email alternative.
This release also fixes translation rerun scope, Outlook CTA alignment, metric
growth, dependency logging, and artifact garbage-collection safety.

No database migration or required configuration change is needed. The new
stage-specific model settings are optional and fall back to the global model.

The release also adds stage-specific LLM settings, provenance-aware refreshes,
