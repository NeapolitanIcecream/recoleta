# Recoleta v0.2.1 Release Notes

Recoleta v0.2.1 is a patch release that adds an operator-side manual trend
email workflow on top of the workflow-first runtime introduced in `v0.2.0`.

## What is in this release

- Manual trend email preview/send under `recoleta run email preview|send` and
  `recoleta fleet run email preview|send`.
- `EMAIL` config plus env-only `RECOLETA_RESEND_API_KEY` for Resend-backed
  delivery.
- Private site email link-map artifacts so email rendering can rewrite
  canonical links to the already deployed public site.
- Batch-oriented send checks that re-render from canonical trend markdown,
  reject mixed partially sent batches unless `--force-batch`, and require the
  primary trend page under `email.public_site_url` to be publicly reachable.

## Suggested release title

`v0.2.1: manual trend email preview and Resend delivery`

## Suggested release summary

Recoleta v0.2.1 keeps the workflow-first and instance-first operating surface
from `v0.2.0`, then adds a manual trend email flow for operators who already
publish a site from canonical trend output. You can preview or send one
selected batch from stored markdown plus sibling presentation sidecars, resolve
links against the deployed site, and send through Resend with batch safety
checks before a resend.
