## Demo outline

### 1. Start from a [Huldra](https://github.com/NeapolitanIcecream/huldra)-backed preset

Show `presets/agents-radar.yaml`, `presets/robotics-radar.yaml`, or
`presets/arxiv-digest.yaml`. Point out that arXiv presets now use
`sources.arxiv.mode: pool` and `arxiv_pool.backend: huldra`.

Suggested line:

`v0.3.0 makes the arXiv-heavy presets pool-first, so the instance run is not responsible for live arXiv metadata fetching.`

### 2. Prewarm and inspect arXiv windows

Show the operator commands:

```bash
uv sync --extra huldra
uv run recoleta arxiv-pool sync --date 2026-01-02 --lookback-days 3
uv run recoleta inspect arxiv-pool freshness --json
```

Suggested line:

`The operator can sync the requested windows first, then inspect pool freshness before spending model calls on analysis.`

### 3. Run with readiness gates

Show a strict workflow using a pool-backed arXiv source:

```bash
uv run recoleta run day --date 2026-01-02 --json
uv run recoleta fleet run day --manifest ./fleet/fleet.yaml --date 2026-01-02 --json
```

Suggested line:

`Strict readiness gates stop downstream analysis, publishing, trends, translation, and site builds when the arXiv pool window is missing or still immature.`

### 4. Check localized output coverage

Show the localization inspector:

```bash
uv run recoleta inspect localization
uv run recoleta inspect localization --json
```

Suggested line:

`After multilingual runs, the localization inspector gives a local coverage check before previewing or publishing the site.`

## Longer launch post

Recoleta v0.3.0 focuses on arXiv-heavy operation. Earlier releases could ingest
arXiv directly from each workspace, but larger fleets need a more deliberate
path: prewarm metadata once, inspect whether the requested windows are ready,
and keep downstream analysis or publishing from running on partial coverage.

This release adds Huldra-backed arXiv pool mode, local SQLite pool commands,
freshness inspection, and workflow readiness gates. A workspace can now use
`sources.arxiv.mode: pool`, configure `arxiv_pool.backend: huldra`, install with
`uv sync --extra huldra`, then run `recoleta arxiv-pool sync` or a fleet
pre-sync before analysis begins.

The readiness checks apply to both single-instance and fleet workflows. They
evaluate the requested date windows, apply a maturity lag, and honor
`readiness_gate: off|warn|strict`. In strict mode, Recoleta can stop before
analysis, publishing, trends, translation, or site build steps depend on a
missing or immature arXiv window.

The same release updates arXiv presets to use Huldra-backed pool mode by
default. It also adds `recoleta inspect localization`, batch-first manual trend
email runs across configured granularities, and workflow timing metrics with
fleet day benchmark helpers.
