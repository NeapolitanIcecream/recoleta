## 60-second demo script

### Beat 1: the operational problem

Recoleta already had the local-first radar model in place, but everyday use
still asked too much from the operator. You had to remember older command
surfaces, special-case multi-instance setups, and piece together run
diagnostics by hand.

### Beat 2: what v0.2.0 changes

v0.2.0 makes the main operating surface workflow-first. The default entrypoints
are now `run`, `fleet`, `inspect`, and `repair`, so the common path is easier
to explain and easier to script.

### Beat 3: fleet and auditability

For multi-instance deployments, Recoleta now uses child configs plus a fleet
manifest. You can run one fleet manually, build an aggregate site, preview it
locally, and inspect recent runs without treating a shared DB as the live
source of truth.

### Beat 4: output quality

Trend and idea outputs now lean on sibling presentation sidecars. That makes
translation, site rendering, and repair flows less dependent on markdown
reconstruction, especially for multilingual notes.

### Beat 5: where to start

If you are new, use the README quickstart. If you are upgrading, start with the
CLI migration guide and the fleet runbook.

## Case-study angles

### 1. Single workspace daily operator

- Hook: `Run one local research radar end to end without remembering legacy subcommands.`
- Show:
  - `recoleta run now`
  - `recoleta inspect runs list --json`
  - `recoleta repair outputs --site --json`
- Main claim:
  `v0.2.0 makes the routine operator path shorter and easier to inspect.`

### 2. Small fleet maintainer

- Hook: `Operate several child instances from one manifest without keeping a shared runtime alive.`
- Show:
  - `recoleta fleet run week --manifest ...`
  - `recoleta fleet site build --manifest ...`
  - `recoleta fleet site serve --manifest ...`
- Main claim:
  `The fleet model is now explicit, previewable, and easier to replay safely.`

### 3. Multilingual publishing flow

- Hook: `Publish localized trend and idea outputs without rebuilding structure from markdown alone.`
- Show:
  - canonical trend or idea note
  - sibling `.presentation.json`
  - localized output tree
- Main claim:
  `Sidecar-first rendering makes translation and site output more predictable.`
