# Recoleta v0.6.0 Release Notes

Recoleta v0.6.0 makes Huldra the default arXiv metadata backend and includes it
in the standard installation. Enabled arXiv sources now use one shared broker
for caching, queueing, request budgets, and durable rate-limit cooldowns.

The release also adds bounded Fleet scheduling over closed UTC windows and
coordinates arxiv.org content requests across Recoleta processes. The old
direct adapter and local SQLite pool remain explicit deprecated rollback paths;
Recoleta never falls back to them automatically.

## What is in this release

- The standard dependency set pins `huldra-arxiv` 0.4.0 to its reviewed merge
  commit. Omitting arXiv mode or pool backend settings now resolves to Huldra
  pool mode for enabled sources.
- `recoleta fleet daemon start --manifest ...` reads schedules from the
  top-level Fleet manifest. Scheduled day, week, and month workflows process
  only closed UTC periods and reconcile at most `catch_up_windows` gaps,
  oldest first.
- A host-local file lease covers each complete Fleet catch-up sequence, so a
  scheduled run and a manual run cannot interleave on one machine.
- Mature day, week, and month windows can request bounded Huldra pre-sync before
  child workflows begin. Page, request, and runtime budgets remain owned by
  Huldra.
- HTML, PDF, and e-print requests use cross-process admission and shared
  cooldown. A 429 does not trigger immediate retries or HTML-to-PDF fallback
  traffic.
- Dry-runs skip arXiv pre-sync and readiness evaluation, so they do not submit
  Huldra work or reach arxiv.org.
- Analyze receipt identity, translation payload ordering, and Outlook title
  spacing receive targeted fixes. Wheel installs also validate the CLI against
  current unlocked dependencies.

## Compatibility and upgrade notes

An enabled arXiv source now requires a reachable Huldra service unless the
operator explicitly selects a deprecated rollback path. Direct mode requires
the `legacy-arxiv` extra. The local SQLite pool requires
`ARXIV_POOL.backend=local_sqlite` and an explicit DB path.

Huldra 0.4.0 advances its store to schema 8. Back up the Huldra database before
the first 0.4.0 startup; normal store initialization performs the additive
migration. Recoleta itself does not add a database migration in this release.

Fleet schedules belong only in the top-level manifest. Run one scheduler per
manifest on one host; the lease is not a distributed lock across machines.

## Suggested release title

`v0.6.0: Huldra-first arXiv automation`

## Suggested release summary

Recoleta v0.6.0 makes Huldra 0.4.0 the installed default for arXiv metadata,
adds bounded Fleet scheduling and mature-window pre-sync, and coordinates
content requests across processes to prevent 429 retry fanout. Deprecated
direct and local SQLite paths remain explicit rollback options with no
automatic fallback. Dry-runs are side-effect free, and targeted fixes cover
analyze receipts, translation payloads, Outlook title spacing, and current
Typer compatibility.
