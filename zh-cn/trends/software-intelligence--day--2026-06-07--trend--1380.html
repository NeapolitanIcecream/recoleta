Claude Code 让递归自我改进成为生产治理问题

概览

当天的信号很窄，但很明确：AI 编码代理已经进入 AI 实验室内部的软件供应链。Claude Code 是最具体的例子，Anthropic 说 Claude 在 5 月写了它发布代码的五分之四以上。

研究发现

AI 实验室里的 AI 生成代码

Claude Code 在这里被当作一个实际可用的软件工程代理，而不是实验室演示。关键事实是内部使用：Anthropic 说它在 5 月发布的代码中，超过五分之四是 Claude 编写的。在 Claude Code 于 2025 年 2 月上线之前，这一比例只有个位数低段。也就是说，在上线后的几个月里，代码生成已经成了公司自己生产流程的一部分。

通过软件工作产生的递归自我改进风险

控制方面的担忧来自软件周围的反馈回路。一个编码代理可以帮助构建开发者工具、基础设施和未来的 AI 系统。如果同一类系统也参与改进或监督后续系统所用的工具，治理就必须覆盖日常工程工作流和模型发布。来源没有提供基准、实验或新的控制方法，所以这些证据支持的是一条风险路径，而不是一项可测量的失控结果。
