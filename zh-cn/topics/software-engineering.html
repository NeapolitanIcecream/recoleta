编码代理运行控制

编码代理上线现在需要在失败成本高的地方设置小型运行控制：混乱的代码库、shell 访问和人工审查。近期最清楚的工作可以被测量：在真实任务上跟踪代理的 token 使用量和文件重复访问次数，把命令执行与长期运行的代理进程隔离，并限制每位审查者同时处理的代理生成 pull request 数量。
