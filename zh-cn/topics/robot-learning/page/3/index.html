Operational Robot Learning

机器人团队可以直接做三项具体改动：把部署 rollout 收集为训练数据，把长时程计划暴露为缓存的文本和关键帧轨迹，再给快速模仿控制器加入空间注意点跟踪。共同的压力来自运行层面：固定的演示集、隐藏的计划和不稳定的视觉特征，会在机器人离开受控测试环境后变成失败。

机器人部署复盘诊断

真实世界的长时程机器人评估已经具体到会改变日常工作流程。最直接的近期动作是按阶段的内部评测、回放后的结构化失败复盘，以及发布门槛里的动态抓取压力测试。证据里的共同点很明确：更大的机器人数据和部署循环，需要更好的方式去看清执行在时间上的断裂点，而不只是看一次运行有没有成功结束。

用于机器人学习的动作-状态对齐
