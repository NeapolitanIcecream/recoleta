Summary

该论文提出 FP-Predictor，用图卷积网络基于代码属性图判断静态安全分析报告是真阳性还是假阳性，目标是减少开发者处理误报的负担。它聚焦 Java 加密 API 误用场景，并在两个基准上展示了较强效果，但结果的一部分依赖人工复核后的标签解释。

Problem

  • 静态应用安全测试（SAST）工具会产生大量假阳性，浪费开发者时间、拖慢真正漏洞修复，并削弱对自动化安全分析的信任。
  • 规则或模式匹配式方法难以理解代码中的语法、控制流和数据依赖等复杂关系，因此难以判断一个报告是否真的可利用。
  • 论文要解决的是：自动预测一条 SAST 报告是误报还是真报，尤其针对 Java 加密 API 误用，这对提升安全扫描工作流效率很重要。

Approach

  • 将每条由 CogniCrypt 报出的漏洞告警，转换为对应方法级别的 Code Property Graph (CPG)，把 AST、CFG、PDG 合并成统一图表示。
  • 用三类节点特征表示代码：Jimple 语句的 Word2Vec 向量、节点类型 one-hot 编码、以及是否为违规节点的二值标记。
  • 使用 GCN 在图上聚合邻居信息，输出一个 �[0,1]� 分数，表示该告警是假阳性的置信度；演示中采用 0.8 阈值，超过即判为假阳性。
  • 在 CamBenchCAP 上训练与测试（431 个标注样本，80/20 划分），再在不同基准 CryptoAPI-Bench 上评估泛化。
  • 论文还对错误样例做人工安全审查，主张部分“误分类”其实反映了模型更保守、偏安全的判断。

Results

  • 在 CamBenchCAP 测试集上，模型报告 100% accuracy（80/20 train-test split）；作者同时承认该数据集规模较小（431 个标注案例）且较结构化，可能高估泛化能力。
  • 在 CryptoAPI-Bench（仅评估 intra-class 场景，排除 21 个 inter-class 案例）上：对真阳性样本，模型正确识别 89/91，误判 2/91 为假阳性，约 97.8% 准确率。
  • 对假阳性样本，按原始基准标签仅正确识别 1/27，即约 3.7%；这说明直接按原标签看，模型对误报识别很弱。
  • 但人工复核后，作者认为最初 26 个“误分类”里有 22 个其实包含真实安全问题或至少是不良加密实践；据此，假阳性子集“有效准确率”从 1/27 (3.7%) 提高到约 23/27 (85.2%)。
  • 基于上述人工重评，论文声称整体准确率可达 96.6%；如果不修改其中 3 个“有争议但属坏实践”的标签，则整体准确率为 94.1%。
  • 训练成本较低：在 Intel i7-11700K + RTX 4090 + 64GB RAM 上训练约 5 分钟。作者还指出当前 CPG 缺少跨过程调用关系，可能导致带 if/control-flow 依赖的案例表现受限。
