Summary

摘要

MuCoCo 是一种自动化测试方法，用来发现代码 LLM 的一致性错误。它把编程任务变形成语义等价的版本，并检查模型在这一对输入上的行为是否发生变化。

问题

  • 代码 LLM 可能会对含义相同的程序给出不同答案，但常见的编程基准主要测试正确性，不测试一致性。
  • 手工构建的基准是静态的，需要人工投入，而且随着时间推移可能泄露到更新的模型中。
  • 一致性很重要，因为不稳定的行为会削弱人们对代码生成、执行预测和其他软件任务的信任。

方法

  • MuCoCo 接收一个编程查询，生成语义等价的变体，分别在原始版本和变异版本上运行 LLM，并比较结果。
  • 它使用 11 种保持语义不变的变异，分为三组：词法、语法和逻辑。例子包括变量重命名、循环改写、德摩根律改写和常量展开。
  • 正确性判定器使用期望输出和测试套件，检查每个输出是正确、错误还是无效。
  • 一致性判定器会在原始查询和变异查询产生不同结果时标记错误，例如通过与失败、有效与无效，或失败的测试用例不同。
  • 评估覆盖 4 个基准、4 种任务类型和 7 个代码 LLM。

结果

  • 在所有设置中，MuCoCo 在生成的测试对里发现 14.82% 存在不一致：21,924 / 147,935 个案例。
  • 模型在这些任务上的平均准确率是 69.13%（102,092 / 147,680），说明即使模型经常答对，不一致仍然很常见。
  • 按模型看，不一致率从 GPT-5 的 2.92% 到 LLaMA-3.1 的 21.22%；GPT-4o 为 18.15%，Gemma-3 为 17.69%，DeepSeek-V3.2 为 16.53%，Codestral 为 14.51%，Qwen2.5 为 12.73%。
  • 按变异类型看，词法变异暴露出最多的不一致，总体为 16.28%（12,974 / 79,707），其次是逻辑变异 13.91%（6,259 / 45,008）和语法变异 11.59%（2,691 / 23,220）。
  • 按任务看，代码生成 的不一致率为 27.15%，MCQ 为 22.35%，输出预测 为 18.99%，输入预测 为 6.65%。
  • 在 Turbulence 数据集上与 Turbulence 对比时，MuCoCo 报告的错误率整体为 41.39%，而 Turbulence 为 33.8%；对应错误数分别是 1,781,288 / 4,302,713 和 279,241 / 826,084。论文称 MuCoCo 在某些设置下效果最高可达 6x，并发现了新的 一致性错误类别。
