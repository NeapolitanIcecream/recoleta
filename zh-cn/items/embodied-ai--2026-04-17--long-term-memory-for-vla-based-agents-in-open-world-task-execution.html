Summary

摘要

ChemBot 面向长时程的化学实验室执行任务，使用一个能够规划、执行、跟踪进度并复用过往成功轨迹的 VLA 智能体。论文的核心主张是：加入双层记忆、闭环子任务规划和具备进度感知能力的 VLA 后，相比 VLA 基线，可以提升任务分解质量和真实世界任务成功率。

问题

  • VLA 系统可以把语言和视觉映射为机器人动作，但它们通常难以处理长流程、依赖状态的实验室协议，因为错误会在多个步骤中不断累积。
  • 现有系统通常将规划与执行分开，也不会持续保存成功运行的记忆，因此机器人会重复试错，而不是复用已经验证有效的方法。
  • 这在化学实验室中很关键，因为任务具有多阶段特征、对安全敏感，并且需要根据当前物理状态在不同子任务之间精确切换。

方法

  • 该系统名为 ChemBot，采用两层设计：基于 LLM 的智能体将任务规划为原子级子任务，名为 Skill-VLA 的 VLA 策略负责在机器人上执行每个子任务。
  • Scene Describer 会把当前实验台图像转换为结构化面板，包含物体、状态、可供性和空间关系，因此规划阶段使用的是有场景锚定的信息，而不只是原始图像。
  • 规划以闭环方式运行：Subtask Reasoner 每次提出一个原子步骤，Reflector 在该步骤加入队列前检查其可行性、冗余性和序列有效性。
  • 记忆系统包含 short-term memory，用于保存当前场景状态、工具输出和任务进度；以及 long-term episodic memory，用于存储以往成功轨迹，供后续实验进行语义检索。
  • Skill-VLA 基于 GR00T，并增加了一个 progress head，用于预测一个位于 �[0,1] 区间的标量子任务完成值。当进度超过阈值时，智能体结束当前技能并启动下一个技能。论文还加入了异步未来状态推理，以减少动作轨迹的不连续性。

结果

  • 在化学任务分解上，在测试的基础 VLM 中，Qwen3-VL-Plus 的文本/结构评分最好，指标为 Edit Distance 0.733、BERTScore F1 0.744、ROUGE-1 0.542、ROUGE-2 0.366 和 ROUGE-L 0.471。Qwen3-VL-Flash 速度更快，总耗时为 98.8 s、每步 16 s，而 Qwen3-VL-Plus 分别为 298.8 s 和 42 s/step。
  • 在使用 Qwen3-VL-Flash 的 ChemBot 消融实验中，full model 达到 Edit Distance 0.766、BERTScore F1 0.694、ROUGE-L 0.324、98.8 s 运行时间和 22,401 tokens。去掉 Scene Describer 对分解质量影响最大，Edit Distance 上升到 0.915，BERTScore F1 降到 0.556。
  • 去掉 Subtask Chain 导致消融实验中最大的结构性退化，Edit Distance 1.195，而完整模型为 0.766。去掉 Reflector 后，Edit Distance 0.881。去掉 Memory 后，token 长度从 22,401 增加到 28,064，这表明记忆模块主要用于降低上下文负载，同时质量与完整模型接近。
  • 用于训练 Skill-VLA 的机器人数据集包含 5,459 expert trajectories，平均 872.2 frames，涵盖 12 类化学流程中的 51,580 atomic action segments。
  • 真实世界评估使用 UR3 机器人、每个任务 16 trials，以及三个多步骤实验：Precipitation (11 steps)、Heat and Dissolution (7 steps) 和 Neutralization (6 steps)。论文称，采用子任务级训练的 Skill-VLA 在这三个任务上都优于完整轨迹基线 (\pi_{0.5}) 和 GR00T。
  • 摘录中没有给出 Fig. 8 的确切成功率百分比，因此目前证据最充分的表述是定性的：完整的 ChemBot 策略在真实世界中的成功率、安全性和精度高于完整轨迹 VLA 基线，作者将这一提升归因于子任务分解和基于进度的技能切换。
