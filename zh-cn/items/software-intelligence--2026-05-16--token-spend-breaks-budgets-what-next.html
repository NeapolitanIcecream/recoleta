Summary

摘要

文章称，AI 编码代理的 token 成本上涨很快，已经迫使许多软件公司做预算决策。这一点重要，因为代理使用正在从开发者工具成本变成可见的工程运营支出。

问题

  • 公司希望工程师使用编码代理，但 token 支出可能比预算和规划周期增长得更快。
  • 昂贵的默认模型、高 effort 设置和没有上限的代理循环，可能把正常编码工作变成每名开发者数百或数千美元的成本。
  • 主要管理问题是判断更高支出何时能换来更快交付，何时属于浪费。

方法

  • 作者采访了 15 家公司的开发者和工程领导者，案例来自大型公司、中型公司、初创公司和种子阶段公司。
  • 文章比较了支出模式、工具选择、默认模型、上限、折扣和管理层应对方式。
  • 核心机制是简单的成本控制：衡量 token 使用量，把简单任务路由到更便宜的模型，设置更便宜的默认模型，限制支出，或在产出增加时接受更高支出。
  • 报道区分了两种常见应对方式：在衡量影响的同时保持开放使用，或通过模型选择和预算限制来控制支出。

结果

  • 两家公司报告称，token 支出在六个月内上涨约 10 倍，且没有明确放缓迹象。
  • 一家种子阶段 AI 基础设施公司报告称，六个月内支出上涨 15 倍，7 名开发者的人均月成本从约 200 美元增至约 3,000 美元。
  • 一家金融科技公司报告称，部分开发者每天在 Claude Code 上花费 500 美元，员工成本实际上翻倍，代码审查成为瓶颈。
  • 一家 2,000 人的 SaaS 公司称，更改默认模型后成本下降了 30%。
  • 一家金融公司在 Cursor 和 Claude Desktop 上各有约 800 至 1,200 名用户，发现每用户 100 美元的额度可能在 3 至 5 个工作日内用完。
  • 其他具体数字包括一次为期一周、金额 1 万美元的缓存错误，高端用户每周约 1,000 美元，一次单日 1,400 美元的 Claude Code 会话，同一工程团队的流量同比增长超过 10 倍，Cursor 在支出约 100 万美元后提供折扣，以及年支出超过 500 万美元的公司报告 Anthropic 没有提供折扣。
