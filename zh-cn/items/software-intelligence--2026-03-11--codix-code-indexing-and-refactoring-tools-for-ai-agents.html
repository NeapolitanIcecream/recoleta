Summary

Codix 是一个面向 AI 编码代理的本地代码符号索引与查询工具，用更精确、低 token 成本的方式替代 grep 式代码搜索。它把代码库解析后存入本地 SQLite，使代理能更可靠地做符号定位、引用查找、关系分析与部分重构相关操作。

Problem

  • AI 代理在大型代码库中导航时，常依赖 grep 或模糊猜测，既耗费 token，也容易误判符号定义、引用和实现关系。
  • 缺少一个对代理友好、可结构化输出、且能随代码变化自动保持新鲜的本地代码索引层，会降低自动化软件开发效率。
  • 这很重要，因为代码智能、自动化软件生产和多代理协作都依赖稳定的程序结构理解，而不是脆弱的文本匹配。

Approach

  • 核心机制很简单：先用 tree-sitter 解析源文件，抽取类、方法、字段等符号及其关系，再把结果存进本地 SQLite 数据库供快速查询。
  • 提供面向代理的命令式接口，如 find、refs、callers、impls，支持按简单名或全限定名进行 glob 匹配，直接返回定义、引用和关系。
  • 每次查询都会基于文件修改时间自动检查并重建过期索引，尽量保证结果始终反映最新代码状态；必要时可用 codix index 全量重建。
  • 输出同时支持紧凑文本和 JSON，前者节省 token，后者方便代理程序化消费；无参数运行也会给出可发现的帮助信息。
  • 当前解析框架声称支持 Go、Java、JavaScript/TypeScript、Python、Rust，但文中也明确说明目前只有 Java 真正可用，其他语言会报出清晰错误。

Results

  • 文本中没有提供正式基准实验或量化评测结果，因此没有准确的速度、准确率、召回率或与基线工具的数值对比。
  • 其最强的具体主张是：通过本地索引替代 grep/find，可让 AI 代理进行更“precise”的符号查找，并减少“burning tokens on grep and guesswork”，但未给出 token 节省百分比或任务成功率提升数字。
  • 系统实现上给出了可验证的能力范围：支持 find、refs、callers、impls 等查询；索引存储在 .codix/index.db；查询时会基于 mtime 自动重建修改过的文件索引。
  • 支持的语言列表为 5 种：Go、Java、JavaScript/TypeScript、Python、Rust；但限制也很明确：当前仅 Java 可用。
  • 索引对象限定为已声明符号（如 classes、methods、fields 等），不支持局部变量和参数；也不检测反射、Javadoc @link/@see 或字符串字面量中的引用。
