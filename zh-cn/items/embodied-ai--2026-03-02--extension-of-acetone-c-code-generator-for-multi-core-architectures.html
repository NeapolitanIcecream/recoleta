Extension of ACETONE C code generator for multi-core architectures

Summary

本文扩展了 ACETONE，将原本面向单核、可认证的 DNN C 代码生成，推进到面向多核 CPU 的可预测并行代码生成。核心贡献是把神经网络推理编排成 DAG 静态调度问题，并用优化后的约束规划/启发式方法在满足安全关键系统可分析性的前提下降低推理时延。

Problem

  • 要把深度神经网络部署到航空等安全关键系统中，不仅要跑得快，还必须可预测、可分析、易认证；单核执行往往推理时间过长，可能无法满足实时约束。
  • 航空嵌入式平台正在从单核转向多核，但短期内未必使用专用加速器，因此需要一种适合多核 CPU 的并行推理代码生成方案。
  • 多核并行化并不简单：既要处理任务划分、核间映射、同步与通信延迟，又要保留 WCET（最坏执行时间）分析能力，这对原本只生成顺序代码的 ACETONE 是直接缺失的能力。

Approach

  • 将固定结构的前馈网络（如 CNN/MLP）表示为 DAG：节点是层，边是数据依赖；节点带 WCET，边带跨核通信延迟，目标是在多核上求静态、非抢占、可重复节点的最短 makespan 调度。
  • 提出一种比 Tang 等方法更高效的约束规划/ILP 风格编码：去掉复杂的 4D 通信变量，改用更少决策变量与改进约束来表达依赖、通信与复制，从而提升求解可扩展性。
  • 引入两类启发式调度：ISH 通过关键路径优先 + 空闲槽插入来快速得到近优解；DSH 进一步尝试复制父节点以减少跨核通信等待，通常更接近最优但更慢、内存开销更高。
  • 在 ACETONE 实现层面，扩展代码生成器以输出裸机多核并行 C 代码，同时自动加入程序到核心的映射和必要的同步机制，并通过 OTAWA 做 WCET 分析、再配合实测验证可预测性。

Results

  • 在随机 DAG（20/50/100 节点，密度 10%）上，ISH 与 DSH 的 speedup 都会随核心数增加而上升后进入平台期；平台由图本身最大并行度决定。文中给出的示例与实验现象表明，更多节点通常带来更高并行收益。
  • DSH 的 speedup 总是高于或等于 ISH，符合其“更接近最优”的设计目标；但它的调度计算时间会随着核心数增加上升 1–2 个数量级，最慢时接近每张图 2 分钟，而 ISH 明显更快且更稳定。
  • 对精确求解而言，作者的改进编码在与 Tang 等方法类似条件下，所有测试配置都至少返回一个解；而 Tang 的表示在给定 1 小时 timeout 下未能在相同条件中及时给出解。作者方法在返回解时，质量至少不差于 Tang 的结果。
  • 在 ILP/CP 求解实验中，平台上的 speedup 在少量核心后也进入平台期；对 20 和 50 节点 数据集，平台值与 DSH 相近，但优化编码大约在 5 核达到平台，而 DSH 约在 7 核 才达到，支持了“DSH 是近优而非最优”的结论。
  • 代价是精确求解非常慢：对 50 节点图，平均求解时间从不低于 54 分钟，且经常触发 1 小时 超时；在较多核心时，DSH 与 ILP 的 speedup 表现相近，但启发式快最多可达 10 倍。
  • 论文摘要还声称扩展方案通过 OTAWA 的 WCET 分析 与实验测得 WCET进行了验证，但给定摘录中没有提供具体 OTAWA 数值或真实网络上的端到端并行代码性能数字。
