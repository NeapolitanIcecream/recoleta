# Flutter 的具体信号是全栈 Dart，GenUI 仍以演示为主

## Overview
当天语料只有一项内容，即一篇 Google Cloud Next 回顾。它提供产品证据，未提供研究结果。最明确的信号是 Dart 通过 Firebase Functions 进一步进入后端工作，同时 Flutter GenUI 通过现场代理驱动界面展示出来。

## Clusters

### 全栈 Dart
Google 预览了 Firebase Functions 的 Dart 支持，因此 Flutter 团队可以用同一种语言编写应用代码和后端函数代码。Dart Admin SDK 也被介绍为一种让更多 Firebase 访问留在 Dart 内的方式。证据来自实用的开发者工具，没有报告延迟、部署、成本或可靠性指标。

#### Evidence
- [That’s a wrap: Everything Flutter at Google Cloud Next](../Inbox/2026-04-27--thats-a-wrap-everything-flutter-at-google-cloud-next.md): 摘要说明了 Firebase Functions 的 Dart 支持预览、Dart Admin SDK 集成，以及缺少量化指标。

### GenUI 和企业证据点
Flutter GenUI 作为面向活动的代理驱动用户界面演示出现。GenLatte 让参会者通过 Flutter GenUI 应用点咖啡，一场会议也介绍了代理创建自身 UI 的内容。同一篇回顾将 Toyota 信息娱乐系统和 Talabat 区域商业工作列为企业案例，但没有给出应用性能或交付速度数字。

#### Evidence
- [That’s a wrap: Everything Flutter at Google Cloud Next](../Inbox/2026-04-27--thats-a-wrap-everything-flutter-at-google-cloud-next.md): 文章正文描述了 GenLatte Flutter GenUI 演示，以及代理式移动和 Web 演示。
