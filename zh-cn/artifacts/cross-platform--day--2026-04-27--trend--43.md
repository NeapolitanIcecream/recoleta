# Flutter 的明确信号是全栈 Dart，而 GenUI 仍以演示为主

## 概览
当天的语料只有一篇 Google Cloud Next 回顾。它提供的是产品证据，不是研究结果。最清楚的信号是 Dart 通过 Firebase Functions 进一步进入后端工作，而 Flutter GenUI 通过现场的 agent 驱动界面展示出来。

## 研究发现

### 全栈 Dart
Google 预览了 Firebase Functions 的 Dart 支持，因此 Flutter 团队可以用同一种语言编写应用代码和后端函数代码。Dart Admin SDK 也被介绍为一种让更多 Firebase 访问留在 Dart 中的方式。证据来自实用的开发者工具，没有报告延迟、部署、成本或可靠性指标。

#### 资料来源
- [That’s a wrap: Everything Flutter at Google Cloud Next](../Inbox/2026-04-27--thats-a-wrap-everything-flutter-at-google-cloud-next.md): 摘要说明了 Firebase Functions 的 Dart 支持预览、Dart Admin SDK 集成，以及缺少定量指标。

### GenUI 和企业证明点
Flutter GenUI 作为面向活动的 agent 驱动用户界面演示出现。GenLatte 让与会者通过 Flutter GenUI 应用点咖啡，另有一场 session 讲了 agents 如何创建自己的 UI。同一篇回顾还把 Toyota 座舱娱乐系统和 Talabat 区域电商工作列为企业示例，但没有给出应用性能或交付速度数据。

#### 资料来源
- [That’s a wrap: Everything Flutter at Google Cloud Next](../Inbox/2026-04-27--thats-a-wrap-everything-flutter-at-google-cloud-next.md): 文章正文描述了 GenLatte Flutter GenUI 演示，以及 agentic mobile 和 web 演示。
