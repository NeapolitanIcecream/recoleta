# Flutter 这一周内容较少，全栈 Dart 是唯一具体信号

## 概览
这一周信息很少。唯一可用的信号来自 Flutter 的 Google Cloud Next 回顾：Firebase Functions 的 Dart 支持预览版，以及 Flutter 中生成式 UI 的演示证据。应把它看作产品方向，因为语料中没有经过基准测试的研究结果。

## 研究发现

### 全栈 Dart
具体的技术公告是 Firebase Functions 的 Dart 支持预览版。核心说法很简单：Flutter 团队可以用 Dart 编写前端代码和后端函数，Dart Admin SDK 则提供更深入的 Firebase 访问能力。来源没有给出延迟、可靠性、部署或成本测量结果，所以这些证据只能支持开发者工作流方向的判断。

#### 资料来源
- [That’s a wrap: Everything Flutter at Google Cloud Next](../Inbox/2026-04-27--thats-a-wrap-everything-flutter-at-google-cloud-next.md): 宣布 Firebase Functions 的 Dart 支持预览版，以及 Dart Admin SDK 集成。

### 生成式 UI 演示
Flutter 的生成式 UI 工作主要通过活动演示呈现，没有经过测量的研究。GenLatte 应用让参会者通过 Flutter GenUI 界面点咖啡，相关会议则把代理驱动界面描述为超越文本聊天的一种方式。Toyota 和 Talabat 被用作企业案例，但这篇回顾没有给出应用性能、发布速度或团队规模数据。
