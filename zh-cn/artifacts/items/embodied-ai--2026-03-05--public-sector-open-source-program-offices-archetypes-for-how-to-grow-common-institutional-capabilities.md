# Public Sector Open Source Program Offices - Archetypes for how to Grow (Common) Institutional Capabilities

## Summary
本文研究欧洲公共部门如何通过开源项目办公室（OSPO）来提升采用、开发与协作开源软件的能力，并提出六类OSPO原型供机构按情境选型。它的价值在于为公共部门建立可复制的组织能力提供了经验化参考。

## Problem
- 公共部门广泛依赖开源软件，但常因外包依赖、采购规则复杂、内部技术能力不足和保守文化，难以系统性利用开源。
- 现有关于OSPO的研究多来自企业场景，缺少面向公共部门、考虑政府层级、资源约束和政策目标的设计指导。
- 这很重要，因为开源被认为支撑**90%+**数字基础设施，并关系到数字主权、互操作性、服务质量、透明度与经济增长。

## Approach
- 采用**定性研究**：访谈**18位**OSPO代表，覆盖**16个**公共部门OSPO案例，地域为欧盟、挪威、列支敦士登和冰岛。
- 对案例进行交叉分析，归纳出**6种OSPO archetypes**，比较其组织结构、职责范围以及如何支持开源采用。
- 用既有产业OSPO文献构建分析框架，关注组织位置、角色设置与职责，如策略、合规、政策流程、社区协作、培训等。
- 通过**2个**后续焦点小组对发现进行验证和补充，并据此形成政策建议。

## Results
- 论文的核心产出是识别出**6种**公共部门OSPO原型，用来指导公共部门根据自身情境、资源和政策目标设计OSPO。
- 实证基础包括**16个案例**、**18位受访者**和**2个焦点小组**，这是文中最明确的研究规模信息。
- 文章声称这些原型可帮助公共部门更好地推动开源采用、共享复用、互操作性、数字主权和政策落实，但**未提供量化性能指标、基准模型或统计显著性结果**。
- 文中还给出若干政策层面的强主张：OSPO应被视为公共部门的战略能力建设机制，而非仅是技术治理单元；其设计需要结合层级、资源、治理目标和本地制度环境。
