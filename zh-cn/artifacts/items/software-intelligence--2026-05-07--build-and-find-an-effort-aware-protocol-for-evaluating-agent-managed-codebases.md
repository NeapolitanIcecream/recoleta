# BUILD-AND-FIND: An Effort-Aware Protocol for Evaluating Agent-Managed Codebases

## Summary
## 摘要
Build-and-Find 评估由一个智能体编写的代码库是否能让后续智能体还原其预期行为和设计选择，以及这种还原需要多少仓库检查成本。

## 问题
- 标准编码智能体基准主要测试代码是否通过行为检查；由智能体管理的仓库还需要向后续智能体传达设计意图。
- 一个仓库可能行为正确，但仍让下游智能体花更多精力找出功能、配置或结构为何这样选择。
- 这会影响多智能体软件工作，因为后续智能体可能把生成的仓库作为主要上下文来审计、扩展或修改代码。

## 方法
- 构建智能体看到隐藏的仓库规格，并创建一个自包含代码库。
- 查找智能体只看到生成的代码库和与规格对应的多项选择题库，然后回答关于预期行为和设计选择的问题。
- 该协议报告经审计的还原准确率、全答对率、可重复性、构建方实现覆盖率，以及新智能体检查字节数。
- 只有通过还原和稳定性检查后，才解释工作量；如果答案不可靠，低检查成本不能算作有力证据。
- 控制分析包括仅问题、仅规格、编译失败、低先验、证据审计，以及构建智能体与查找智能体亲和性分析。

## 结果
- 发布的研究使用 2 个仓库任务族、每个任务 15 个问题、48 次构建、1,728 条基于产物条件的查找记录，以及 25,920 行原始问题记录；编译通过的主面板包含 41 次构建、1,476 条查找记录，以及 21,312 行已评分的查找智能体答案。
- 仅问题准确率为 94.5%，仅规格准确率为 99.9%，编译通过且基于产物条件的还原准确率为 98.9%，比仅问题高 4.4 个百分点。
- 使用产物时的可重复性高于仅问题控制：正式的基于产物条件的 3 次试验一致率为 97.5%，可靠问题占比为 96.7%；仅问题对应数值为 86.7% 和 66.7%。
- 构建方实现覆盖率范围为 85.0% 到 100.0%，其中四个构建智能体达到 100.0%；对已实现黄金声明的下游还原准确率为 97.3% 到 99.5%。
- 在低先验子集上，仅问题准确率为 88.9%，基于产物条件的还原准确率为 97.9%，提升 9.0 个百分点；scratch_minidb 从 90.0% 升至 96.8%，scratch_nanoweb 从 87.8% 升至 98.9%。
- 条件工作量将 GPT-5.5 产物排为低成本、全覆盖示例；在高工作量面板中 R_b = 1.033，GPT-5.5-low 的 R_b = 1.151；GPT-5.4-mini-high 在高工作量面板中排名第二，R_b = 1.320。
