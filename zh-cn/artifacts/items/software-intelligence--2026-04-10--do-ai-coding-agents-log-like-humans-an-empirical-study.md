# Do AI Coding Agents Log Like Humans? An Empirical Study

## Summary
## 摘要
这篇论文研究 AI 编码代理在真实开源 pull request 中如何处理软件日志。结果显示，代理修改日志的频率低于人类，经常忽略明确的日志指令，并把许多生成后的日志修复工作留给人类完成。

## 问题
- 日志是用于调试和运行软件的可观测性核心要求，但此前的工作没有展示 AI 编码代理在真实代码仓库中如何处理它。
- 团队需要知道代理是否遵循人类的日志习惯、是否响应日志指令，以及它们会减少还是增加维护工作。
- 日志不足会让系统更难诊断，而日志过多会增加噪声和开销。

## 方法
- 作者分析了来自 AIDev 数据集的 **81 个开源代码仓库**中的 **4,550 个代理 PR** 和 **3,276 个人工 PR**；在排除双方都没有日志变更的代码仓库后，主要分析使用 **77 个代码仓库**。
- 他们使用按语言定制的正则规则，在 Python、Java 和 JS/TS 的 diff 中检测日志变更，并排除生成产物以及测试/构建噪声。对 **380 个 diff** 的人工检查报告 **96% precision** 和 **94% recall**。
- 他们在每个代码仓库内比较代理 PR 和人工 PR，指标包括日志变更出现率、日志密度、消息特征、日志级别和语法位置。
- 他们从关联 issue、代码仓库指令文件和 PR 审查评论中收集代理指令，然后用 GPT-4o、GLM-4.7 和 DeepSeek-V3.2 组成的三模型投票方案对日志相关意图进行分类。在 **100 个人工标注样本**上，提示词迭代后达到 **Cohen's kappa = 0.83**。
- 他们还追踪生成后的日志调整，以判断后续的日志修复是由人类还是代理完成。

## 结果
- 在 **77 个代码仓库中的 45 个（58.4%）**里，人类修改日志的频率高于代理。配对差异在统计上显著，**p = 0.019**。标准化分数的中位数是 **0.45**，作者将其解释为：在一个典型项目中，代理修改日志的频率比人类大约低 **16%**。
- 在双方都会添加日志的代码仓库中，代理每 **1,000 modified LOC** 引入的日志数量比人类高 **30%**，因此代理虽然在 PR 层面记录日志的次数更少，但一旦记录，密度可能更高。
- 明确的日志指令很少：被研究的指令来源中只有 **4.7%** 包含这类指令。
- 对于建设性的日志请求，代理有 **67%** 的时间没有遵从。论文还指出，即使指令很具体，这种低遵从率仍然存在。
- 生成后的日志修复中，有 **72.5%** 由人类完成，通常是通过后续提交而不是明确的审查评论完成，这说明存在隐藏的维护负担。
- 论文还声称，代理在错误日志模式上比在信息性上下文日志上更接近人类，但摘录没有提供关于日志级别或语法上下文的详细数值拆分。
