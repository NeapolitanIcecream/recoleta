# Uber blows through its AI budget in 1 quarter

## Summary
## 摘要
据报道，Uber 在四个月内用完了 2026 年 AI 编程工具预算。公司首席运营官表示，Uber 目前还无法把 Claude Code 的使用量和更多有用的消费者功能联系起来。文章把 AI 编程代理描述为企业的成本控制问题，尤其是在按 token 计费的使用量上升时。

## 问题
- Uber 提高了 Claude Code 的使用，但 Andrew Macdonald 说，公司无法清楚地把使用数据和推出更多有用的消费者功能直接对应起来。
- 这个成本问题很重要，因为代理式编程工具每项任务会消耗很多 token，所以即使模型推理变便宜，总支出也可能因为采用率上升而增加。
- 内部激励机制可能会推动工具使用，但不能证明产品影响，Uber 用 AI 工具总使用量给团队排名的排行榜就是例子。

## 方法
- 这是一篇商业新闻，不是研究论文，所以没有提出新的技术方法。
- 文中描述的核心机制很直接：员工使用 Claude Code 和类似的代理式工具；这些工具在帮助编程或完成工作任务时会消耗 token；按使用量计费会把更多工具活动变成更高账单。
- Uber 通过一个按 AI 工具总使用量排名的内部排行榜鼓励采用，这可能在价值指标还不清楚时就提高了消耗。
- 文章把 Uber 的情况和 Microsoft 据报道把工程师从直接使用 Claude Code 许可证转向 GitHub Copilot CLI 作了对比。

## 结果
- 据报道，Uber 在四个月内用完了 2026 年 AI 编程工具的全部预算。
- Uber 首席执行官 Dara Khosrowshahi 说，公司已提交代码中约 10% 是由自主代理生成的。
- Macdonald 说，Uber 目前还无法证明 Claude Code 的使用会带来类似“25% 更多有用的消费者功能”的结果。
- Gartner 预计，到 2030 年，先进 AI 模型的推理成本会比 2025 年低 90%，但也表示企业 AI 的成本未必会下降，因为代理式模型会使用更多 token，而且供应商可能会保留一部分节省下来的成本。
- Gartner 预测，AI 代理软件支出将在 2026 年达到近 2070 亿美元，比 2025 年的 864 亿美元增长超过 139%。
- 据文章称，Uber 在 2026 年第一季度的研发支出为 9.51 亿美元，比一年前同期高近 17%；2025 年的研发支出比 2024 年高 3.4%。
