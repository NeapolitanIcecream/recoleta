# That’s a wrap: Everything Flutter at Google Cloud Next

## Summary
## 摘要
这是一篇 Google Cloud Next 活动回顾，不是研究论文。文章宣布 Firebase Functions 的 Dart 支持预览版，并展示 Flutter 在 GenUI、企业应用和开发者社区活动中的工作。

## 问题
- Flutter 开发者从应用代码转到后端函数时，常常需要切换语言和工具，这会给全栈项目增加摩擦。
- 当用户需要选择、审核或处理生成内容时，基于智能体的应用需要文本聊天以外的 UI 界面。
- 企业团队需要证据来判断 Flutter 能否支持大型产品界面，包括汽车和区域电商应用。

## 方法
- 核心产品动作是全栈 Dart：用 Dart 编写 Flutter 应用代码和后端 Firebase Functions 代码。
- Dart Admin SDK 为 Dart 开发者提供更深入的 Firebase 访问能力，因此更多后端工作可以留在同一种语言中完成。
- Flutter GenUI 通过演示展示，其中智能体创建或驱动 UI，包括 GenLatte 咖啡点单应用。
- 活动通过现场演示、会议环节以及 Toyota 和 Talabat 的客户案例展示生产用例。

## 结果
- 文章没有报告定量研究结果、数据集、基线或基准比较。
- Google Cloud Next 在拉斯维加斯有超过 30,000 名参会者，这让这些公告触达了大量开发者。
- 主要技术公告是 Firebase Functions 的 Dart 支持预览版；文章没有给出延迟、成本、可靠性或部署指标。
- GDE Summit 在主活动前聚集了超过 350 名 Google Developer Experts。
- Toyota 被作为使用 Flutter 构建下一代信息娱乐系统的案例提及，Talabat 被作为在中东扩展产品的案例提及；文章没有给出应用性能、团队规模或发布速度数据。
