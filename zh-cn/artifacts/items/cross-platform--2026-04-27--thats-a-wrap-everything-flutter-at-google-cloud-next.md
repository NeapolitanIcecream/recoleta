# That’s a wrap: Everything Flutter at Google Cloud Next

## Summary
这是一篇 Google Cloud Next 活动回顾，不是研究论文。它宣布了 Firebase Functions 的 Dart 支持预览，并展示了 Flutter 在 GenUI、企业应用和开发者社区活动中的工作。

## Problem
- Flutter 开发者在应用代码和后端函数之间切换时，常常要更换语言和工具，这会给全栈项目带来摩擦。
- 当用户需要选择、审阅或处理生成内容时，基于代理的应用需要的不只是文本聊天界面。
- 企业团队需要证据，证明 Flutter 可以支持大型产品界面，包括汽车和区域电商应用。

## Approach
- 核心产品动作是全栈 Dart：用 Dart 编写 Flutter 应用代码和后端 Firebase Functions 代码。
- Dart Admin SDK 为 Dart 开发者提供更深入的 Firebase 访问，让更多后端工作可以留在同一种语言里完成。
- Flutter GenUI 通过演示展示，代理可以创建或驱动界面，包括 GenLatte 咖啡点单应用。
- 这次活动通过现场演示、会议和来自 Toyota 与 Talabat 的客户案例展示生产场景。

## Results
- 没有报告定量研究结果、数据集、基线或基准对比。
- Google Cloud Next 在拉斯维加斯吸引了超过 30,000 名参会者，这让这些公告触达了大规模开发者受众。
- 最主要的技术公告是 Firebase Functions 的 Dart 支持预览；文章没有给出延迟、成本、可靠性或部署指标。
- GDE Summit 在主活动前汇集了超过 350 位 Google Developer Experts。
- Toyota 被提到在下一代车载信息娱乐系统中使用 Flutter，Talabat 被提到在中东地区扩展产品；文中没有给出应用性能、团队规模或发布速度的数据。
