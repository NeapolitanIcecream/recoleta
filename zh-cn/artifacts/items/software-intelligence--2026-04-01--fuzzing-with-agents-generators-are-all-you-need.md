# Fuzzing with Agents? Generators Are All You Need

## Summary
## 摘要
Gentoo 使用一个 LLM 编码代理，基于源代码和可选的谓词级反馈来编写面向特定目标的模糊测试输入生成器。在 7 个 Java 基准上，论文称这些由代理编写的生成器在分支覆盖率上通常优于人工编写的生成器，并且基本不再需要覆盖引导的变异。

## 问题
- 面向结构化输入的模糊测试器经常失效，因为随机输入或轻度变异后的输入无法满足到达程序深层逻辑所需的语法和语义约束。
- 人工编写的领域专用生成器可以解决这个问题，但需要大量手工工作，而且通常只适用于单一目标。
- 论文要回答的是，AI 编码代理能否自动写出这类足够强的生成器，从而减少或去掉对覆盖引导和变异的需求。

## 方法
- Gentoo 向一个 LLM 编码代理提供终端访问权限、模糊测试目标以及库的源代码，然后要求它迭代编写和改进 JQF 输入生成器。
- Gentoo-S 加入静态谓词排序：它构建过程间控制流图，用 WALA 计算分支支配分数，并告诉代理哪些分支结果控制了最多代码。
- Gentoo-L 用代理基于源代码检查产生的谓词排序替代静态分析。Gentoo-Base 不使用谓词引导。
- 在模糊测试过程中，Gentoo-S 和 Gentoo-L 会收集每个谓词的分支计数，因此代理可以看到哪些高价值分支很少被走到，并为这些条件补充生成逻辑。
- 核心想法很直接：检查程序，推断什么样的输入结构和语义能打开那些难以到达的分支，然后把这些规则直接写进生成器，而不是依赖变异去碰到它们。

## 结果
- 评估覆盖 **7 个真实世界 Java 库**，并将 **3 个 Gentoo 变体** 与 **人工编写的生成器** 进行比较。
- 代理合成的生成器在 **7 个基准中的 6 个** 上取得了高于人工编写生成器的分支覆盖率，其中 **4 个基准** 的提升 **具有统计显著性**。
- 相比人工编写的生成器，论文报告的平均覆盖率提升为 **11% 到 21%**，具体取决于配置。
- 对于代理合成的生成器，与随机生成相比，使用覆盖引导变异通常只带来 **低于 3%** 的改进，论文称这些差异 **不具有统计显著性**。
- 对于人工编写的生成器，论文报告覆盖引导在所有基准上都 **具有统计显著性**，这支持了一个说法：Gentoo 生成器已经编码了足够的结构和语义信息，所以变异带来的额外收益很小。
- 谓词引导的细化相较于各方法的第一次迭代，平均将覆盖率提高了 **18% 到 57%**，分别对应 **Gentoo-S** 和 **Gentoo-L**，不过论文也表示 Gentoo-Base、Gentoo-S 和 Gentoo-L 的最终覆盖率整体上差异不大。
