# Applying an Agentic Coding Tool for Improving Published Algorithm Implementations

## Summary
## 摘要
这篇论文提出了一个两阶段工作流：先用一个模型寻找合适的近期论文，再用 Claude Code 复现并改进这些论文公开发布的实现。在 11 个已发表算法实现上，作者报告称每个案例都在一个工作日内取得了改进，但他们也说明，这些结果没有在代理环境之外得到独立验证。

## 问题
- 现在很多研究论文都会公开代码和数据，但关于代理式编程工具能否接手一个已发表的实现、完成复现，并把结果提升到论文报告的基线之上，相关证据仍然很少。
- 这很重要，因为已发表的实现构成了一个规模很大的具体且可运行的目标集合，可用于 AI 辅助研究、复现和渐进式算法改进。
- 论文也在问：当代理能够搜索、写代码、运行实验并记录迭代过程时，哪些工作仍然需要人来完成。

## 方法
- 这个工作流分为两个阶段：发现阶段和改进阶段。
- 在发现阶段，ChatGPT Deep Research 搜索满足明确筛选条件的论文：GitHub 上有 Python 或 C++ 代码、数据集公开、在至少两个数据集上的运行可在约 30 秒内完成、发表于 2021 年之后，且发表在 Q1 或 Q2 期刊。
- 在改进阶段，Claude Code 阅读论文 PDF，选择一个指标-数据集组合，复现基线，然后对实现进行最多 20 次迭代，同时保存每次运行的代码、结果和简短计划。
- 一旦在选定目标上超过论文报告的结果，代理就会停止。日志文件的作用是让每个实验都可审计。
- 研究将这一流程应用到 11 个领域，包括组合优化、可解释 AI、模式挖掘、图像分割、数据流处理、分布式系统、网络安全、图机器学习、分子模拟、计算物理和生物信息学。

## 结果
- 主要结论是，11 个实验中 11 个都相对于选定的已发表基线取得了改进，而且每个报告中的改进都在一个工作日内完成。
- 报告中的量化提升包括：组合优化中运行时间提升 193 倍，模式挖掘中运行时间提升 6.4 倍，在高 K 图像分割任务中以全局最优实现超过 1000 倍加速，分布式系统中查询延迟降低到原来的二分之一以下，以及生物信息学中典型情况 10.5 倍、大规模实例 34.3 倍的加速。
- 其他报告的提升包括：图机器学习中准确率提高 8%，网络安全中防御成功率翻倍以上，数据流处理中在 10 KB 条件下更高的 F1，可解释 AI 中更低的稀疏度，以及分子模拟中在所有测试取值下更低的动力学温度误差。
- 论文也给出了重要限制：作者没有逐行独立审查生成的代码，没有在 Claude Code 环境之外重新运行结果，并且将模型报告的输出视为初步结果。
- 有些对比没有看起来那么强。网络安全实验使用了 50 个基准网络，而论文原本用了 200 个，并且因为原始引擎是专有系统，所以改用了替代模拟器。在图机器学习中，这一提升来自替换原始方法，而且代理实际运行了 38 次迭代，尽管提示中将上限设为 20 次。
- 论文更广泛的结论是，人类工作仍然重要，包括选择目标、检查实验是否有效、判断新颖性和影响、提供算力和访问权限、管理执行风险，以及撰写披露说明。
