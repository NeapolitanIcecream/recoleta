# Leap Second

## Summary
闰秒是为了让基于原子钟的UTC持续贴近地球自转定义的太阳时而引入的1秒校正机制。本文系统说明了闰秒的物理原因、实施规则、工程问题，以及到2035年前后逐步取消闰秒的国际进展。

## Problem
- 需要解决**原子时间（TAI）**与**地球自转时间（UT1）**之间不断累积的偏差，否则民用时间UTC会逐渐偏离太阳时。
- 闰秒虽然保持UTC与UT1的差值不超过约**±0.9秒**，但其**不规则、仅提前约6个月宣布**，给计算、网络同步、导航和交易系统带来实现复杂性与故障风险。
- 不同系统对闰秒的处理不一致（重复秒、23:59:60、冻结、smear等），导致跨系统时间戳不一致，影响精确排序、持续计时和互操作性。

## Approach
- 核心机制很简单：当**UTC快要与UT1偏差过大**时，由**IERS**宣布在某个月末插入或理论上删除**1秒**，把UTC重新拉回接近UT1。
- 自**1972年**起，UTC按严格SI秒运行，但在需要时于**23:59:60**插入正闰秒；负闰秒理论存在，但文中指出**尚未发生**。
- 规则上，IERS通常在偏差接近**0.6秒**时决定操作，以确保**UTC-UT1不超过±0.9秒**；公告通常每**6个月**发布一次，优先在**6月末或12月末**执行。
- 文中还讨论替代方案：用**TAI**做内部时间基准、采用**leap smear**平滑调整、或未来用更少见的**leap minute / leap hour**替代频繁闰秒。
- 国际标准层面，**2022年CGPM Resolution 4**决定在**2035年或之前**放宽UT1-UTC允许差值，实质上为停止常规闰秒铺路，同时保留UTC与地球自转的长期联系。

## Results
- 自**1972年**以来，UTC共加入**27个正闰秒**；最近一次发生在**2016-12-31**。截至**2024年**，**TAI与UTC相差37秒**（初始10秒 + 27个闰秒）。
- **1972–2020**期间，闰秒平均约每**21个月**一次，但分布高度不规则：例如**1999-01-01到2004-12-31**间隔**6年**没有闰秒，而**1972–1979**的**8年**里有**9次**闰秒。
- 地球平均日长在过去几个世纪每世纪约增加**1.4–1.7毫秒**；潮汐摩擦单独贡献约**2.3毫秒/世纪**，但地球内部与表面质量再分布会抵消一部分。
- 文中给出极端观测：**2022-06-29**是记录中最短一天，比24小时短**1.59毫秒**；**2020年**出现了自1960年以来**28个**最短日，推动了对**负闰秒**与替代方案的讨论。
- 国际决策方面，**2022年**CGPM已通过在**2035年或之前**提高UT1-UTC上限的决议，**2023年WRC-23**正式认可该方向；文本未给出像论文实验那样的统一benchmark指标，而是以历史统计、制度阈值和大量工程事故案例作为主要“结果”。
