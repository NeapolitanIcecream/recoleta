# Figures as Interfaces: Toward LLM-Native Artifacts for Scientific Discovery

## Summary
## 摘要
论文提出了**LLM 原生图形**：科学图形自身携带数据、代码、来源信息和交互映射，因此 LLM 可以把图形当作分析对象来操作，而不只是把它看成像素。论文还提出了 **Nexus**，这是一个用于科学计量学领域迭代式科学探索的概念验证系统。

## 问题
- 科学图形通常是静态输出。它们向人展示模式，但会隐藏生成这些图形所用的数据子集、转换步骤和代码。
- 在当前的人机协作流程中，多模态 LLM 往往需要从图像或图注中还原含义，这会削弱可复现性、可追溯性和后续分析能力。
- 科学探索是迭代且非线性的，但许多 LLM 研究工具仍以线性对话或一次性生成为中心，这使分析步骤难以细化、审计和复用。

## 方法
- 核心思路是把每个图形存储为结构化对象 \(F_t = \{V_t, C_t, D_t, M_t\}\)：可视化输出、可执行代码和分析操作、底层数据，以及元数据/历史记录。
- 系统在视觉标记与分析操作之间保留**双向映射**。语言请求可以生成图形，而对图形的直接交互，如框选点或时间范围，也可以映射回精确的数据行，并生成下一步分析。
- 论文将一组关联图形及其历史组织成**数据驱动工件**，其中记录用户输入、派生图形、代码、数据，以及跨步骤的协调规则。这个工件以 DAG 方式进行版本管理，因此可以回到并重放先前状态。
- LLM 在受限动作空间上运行计划-行动-观察循环，这些动作包括过滤、转换、建模和可视化，然后执行代码，并根据输出决定下一步。
- Nexus 将这一设计实现到科学计量学分析中，用户可以在探索过程中把自然语言指令与图形操作结合起来使用。

## 结果
- 论文认为主要有三点收益：更快的迭代发现、更强的可复现性，以及通过保存来源信息和可执行历史带来的更透明推理。
- 论文给出了一个科学计量学领域的案例研究，使用 Nexus 探索美国一所大型研究型大学的发明人层面创新数据，并进行交互式后续修改，如对坐标轴做对数缩放，以及深入查看选定区域。
- 论文还表示，系统通过计算测试评估了**图形与分析操作之间双向映射的保真度**。
- 给出的摘录**没有包含准确率、速度或用户结果的定量指标或基准数字**，因此无法从现有文本中提取相对基线的数值提升。
- 摘录中的具体技术主张包括：除非分析过程本身是非确定性的，否则图形可以根据保存的代码和数据被确定性重建；关联图形也可以通过预计算的协调关系和代码模板自动更新。
