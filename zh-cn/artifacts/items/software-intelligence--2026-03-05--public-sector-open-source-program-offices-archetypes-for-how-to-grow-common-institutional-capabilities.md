# Public Sector Open Source Program Offices - Archetypes for how to Grow (Common) Institutional Capabilities

## Summary
本文研究欧洲公共部门如何通过开源项目办公室（OSPO）来系统提升采用、开发和协作开源软件的能力，并提出六类OSPO原型作为组织设计参考。它的核心价值在于把分散的公共部门开源实践整理成可复用的制度模式，用于支持数字主权、互操作性和公共服务质量提升。

## Problem
- 公共部门高度依赖开源软件，但在采购、合规、技术能力、组织文化和长期政策支持方面常缺乏系统化能力，导致难以有效利用开源。
- 现有关于OSPO的研究多来自企业场景，缺少针对公共部门、不同政府层级、资源约束和政策目标的设计指导。
- 这个问题重要，因为开源软件支撑**超过90%**的数字基础设施，公共部门若缺乏相应能力，将影响数字化转型、互操作性、透明度与数字主权。

## Approach
- 作者采用**定性研究**：访谈**18位**OSPO代表，覆盖**16个**欧洲公共部门OSPO案例，范围包括欧盟、挪威、列支敦士登和冰岛。
- 通过跨案例分析，将这些实践归纳为**6种OSPO原型**，比较它们的组织结构、职责范围以及支持开源采用的方式。
- 研究还用**2场**后续焦点小组对发现进行验证和补充，参与者包括先前受访者和额外专家。
- 核心机制可以简单理解为：先系统观察多个公共部门OSPO怎么运作，再把共性模式抽象成“原型”，帮助其他机构按自身情境选型和设计。

## Results
- 论文的主要成果是识别出**6种**公共部门OSPO原型，并给出与组织结构、职责和政策目标相关的设计洞见。
- 研究样本包括**18位受访者 / 16个案例 / 2场焦点小组**，这是文中最明确的实证规模信息。
- 论文声称这些原型可为公共部门组织提供可操作的设计参考，帮助其根据**政府层级、资源条件、政策目标**来配置OSPO。
- 论文还提出政策建议，强调OSPO可作为促进开源采用、数字主权、经济增长、互操作性和数字服务质量的战略工具。
- 文本摘录**未提供定量性能指标**，也没有给出与基线方法的数值对比；其最强的具体主张是：该研究首次基于欧洲公共部门案例，形成了面向实践的**六类原型化分类**与政策建议。
