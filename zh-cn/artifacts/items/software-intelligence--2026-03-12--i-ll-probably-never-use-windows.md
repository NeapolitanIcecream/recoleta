# I'll probably never use Windows

## Summary
这是一篇观点性文章，批评 Windows 正从“本地、可控的个人操作系统”转向“云端绑定、广告化、可监控的平台”。作者据此表示自己大概率不会再使用 Windows，并主张更重视用户自主权与隐私。

## Problem
- 文章关注的问题是：Windows 的产品方向正在削弱用户对自己电脑的控制权，把操作系统变成更依赖云、账号和厂商策略的服务。
- 这之所以重要，是因为它涉及隐私、可靠性和自主性：更新失误可能让电脑不可用，强制账号和云集成可能扩大追踪与审查能力。
- 作者还认为，大众常把操作系统等同于“电脑本身”，因此往往难以及时意识到这些结构性变化带来的长期风险。

## Approach
- 这不是学术论文或实验研究，而是一篇基于历史版本对比与个人判断的评论文章：将 Windows 7 之后的 Windows 8/10/11 发展轨迹作为核心论据。
- 核心机制的简单说法是：作者把微软近年的策略概括为“Windows 从本地工具变成持续联网、持续更新、持续绑定账号的服务”。
- 文中用几个具象现象支撑这一判断：强推 Microsoft 账号登录、更多云迁移、在系统中加入广告与 AI、减少 QA 并把用户当测试对象、更新导致设备异常。
- 在替代方案上，作者提出 Linux 更适合重视隐私的人；macOS 若已拥有 Mac，则是相对更精致但仍非首选的替代品。

## Results
- 没有提供严格的定量实验、数据集、指标或基线比较；这是一篇规范性/评论性文本，而非可复现实证研究。
- 最强的具体主张之一是：Windows 10 和 11 的一些更新“让许多 PC 无法使用”，但文中没有给出数量、时间范围或来源数据。
- 另一个具体结论是：作者认为 Windows 7 是“最后一个真正优秀的 Windows 版本”，而 Windows 8/8.1 开始明显转向平板式界面与账号绑定，Windows 10/11 则进一步服务化、广告化、云化。
- 关于替代方案，文中给出的可验证程度较高的现实判断是：由于 Proton 的存在，Linux 上的 Steam 游戏兼容性“显著改善”；但同样没有给出兼容率数字或基准比较。
- 最终结论不是性能突破，而是立场性判断：如果用户重视隐私、设备所有权和本地控制，作者认为继续使用 Windows 的吸引力正在下降。
