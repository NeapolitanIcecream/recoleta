# ClawNet: Human-Symbiotic Agent Network for Cross-User Autonomous Cooperation

## Summary
## 摘要
ClawNet 提出了一种跨用户智能体系统，其中每个智能体都绑定到一位人类所有者，并可在明确的身份、权限和审计规则下与其他用户的智能体协作。论文的核心观点是，当前的智能体系统已经较好地支持了单用户自动化，但缺少多用户自主协作所需的治理机制。

## 问题
- 当前的单智能体和多智能体系统大多为一个用户或一个共享主体服务，因此不支持智能体代表不同的人、围绕各自不同的利益行事。
- 跨用户自主协作需要三个当前缺失的要素：稳定的所有者身份、范围受限的授权，以及动作级问责。
- 缺少这些控制时，跨用户智能体会带来未授权访问、私人数据泄露，以及在缺乏明确人工授权的情况下作出决策的风险。

## 方法
- ClawNet 构建了一个**人机共生智能体网络**，网络节点是人类，智能体在他们之间充当协作中介。
- 每个用户都有一个永久绑定的智能体系统，其中包括一个**管理智能体**和多个**身份智能体**。管理智能体掌握广泛的所有者知识，但与外部通信隔离；身份智能体只暴露特定上下文中的角色和知识。
- 跨用户协作需要双方批准、身份智能体之间的授权可见性，以及在必要时向其他用户进行有边界的递归委托。
- 系统对每个动作都实施三种治理机制：将身份绑定到特定所有者和身份智能体、对允许资源进行范围受限的授权，以及仅追加的审计日志。
- ClawNet 采用云边设计：持久化云服务负责编排和身份状态，本地客户端执行操作系统级文件操作，并提供两道独立的权限检查、备份和回滚支持。

## 结果
- 这段摘录**没有提供基准表或汇总性的定量评估指标**，例如准确率、成功率、延迟或成本。
- 论文声称，在其对比表中，ClawNet 是第一个同时完整支持以下四项属性的系统：**身份绑定、范围受限的授权、动作问责和跨所有者协作**；其他类别被标记为部分支持或不支持。
- 该系统通过本地节点端点支持**操作系统级**文件操作，例如读取、写入、移动、重命名、复制、创建和安全删除。
- 授权路径有**两层相互独立的机制**：服务端 L1 ACL 和客户端 L2 文件夹白名单；任意一层拒绝请求，系统都会以失败关闭方式拒绝执行。
- 在文中描述的采购案例中，工作流经过了**9 个步骤**的智能体间协商与升级，最终决策需要人工批准，未授权请求会被治理检查拒绝。
- 作者表示，ClawNet 已经**部署并在跨组织协作场景中测试**，并声称带来了几项具体效果，包括身份隔离、授权执行、完整可审计性、执行前备份，以及针对文件变更操作的撤销或批量回滚。
