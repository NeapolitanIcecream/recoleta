# The Effect of Code Obfuscation on Human Program Comprehension

## Summary
这篇论文通过受控人类实验研究代码混淆如何影响程序理解，使用输出预测任务测量准确率与作答时间。结论是：混淆通常会让人更慢且更不准，但其影响并非总是随“混淆强度”单调增加，并且在 Python 与 JavaScript 上表现不同。

## Problem
- 论文要解决的问题是：**代码混淆到底如何影响人类理解程序**，而不仅是影响自动化指标；这很重要，因为混淆的真实目标本来就是阻碍人类分析、逆向和理解代码。
- 现有常见假设是“混淆越强，人越难懂”，但作者指出这未必对人类认知成立；不同混淆方式（重命名、误导性命名、控制流改写、组合）可能作用于不同认知线索。
- 还要回答跨语言与经验迁移问题：Python/JavaScript 中混淆效果是否不同，以及某一语言的经验是否能迁移到另一语言的混淆代码理解中。

## Approach
- 作者设计了一个**输出预测**的人类实验：给参与者一段函数代码和输入，让其写出精确输出；这样可以用客观的正确/错误与响应时间衡量程序理解。
- 构建 5 个混淆层级：L0 原始代码，L1 无语义标识符重命名，L1b 对抗性误导命名，L2 控制流改写/扁平化，L3 重命名+控制流组合。
- 数据来自 **HumanEval-X** 的 Python 与 JavaScript 子集；每种语言随机选 **10** 个函数片段，限制圈复杂度 **4–8**、代码行数 **<15**，并保证各混淆版本语义等价。
- 进行线下受控实验：**50** 名计算机专业本科生，每人 **75 分钟** 内完成 **12** 道题；记录正确率、每题耗时、以及自报的 Python/JavaScript 经验。
- 用双系统理论解释现象：未混淆代码更容易触发快速启发式（System 1），而混淆会迫使参与者转向更慢、更刻意的逐句追踪（System 2）。

## Results
- 整体上，未混淆代码 **L0 准确率最高 40.46%（53/131）**；最强组合混淆 **L3 最低 31.09%（37/119）**。相对 L0，**L1 -1.78%**、**L1b -2.44%**、**L2 -6.31%**、**L3 -9.37%**，说明控制流混淆比单纯重命名更伤害理解。
- 各层准确率分别为：**L0 40.46%（53/131）**、**L1 38.68%（41/106）**、**L1b 38.02%（46/121）**、**L2 34.15%（42/123）**、**L3 31.09%（37/119）**。作者据此声称：人类对重命名相对更有韧性，但对控制流扰动更脆弱。
- 全部作答共 **600** 次，整体正确率仅 **36.5%**，说明即使没有混淆，任务本身也不简单；混淆是在一个已具挑战性的理解环境上进一步“加税”。
- 响应时间方面，**L0 平均 140 秒**；一旦加入混淆（**L1–L3**），平均作答时间会**增加约 1 分钟**并维持在高位，表明参与者从快速模式转向更费力的分析模式。
- 在“快答 vs 慢答”比较中，**L2 的慢答相对快答准确率差距最大，为 Δ15.1%**，说明控制流混淆尤其会误导快速启发式判断，但花更多时间逐步追踪还能部分恢复表现。
- 正确与错误答案的时间分布显示一种“适度思考最优”模式：正确答案中位数 **142.7 秒**，错误答案中位数 **127 秒**；但极长时间常对应困惑而非更高准确率。论文还指出在 **L3** 下即便慢答组准确率也只有 **34.5%**，接近基线 **32.4%**，表明组合混淆会压垮理解。
