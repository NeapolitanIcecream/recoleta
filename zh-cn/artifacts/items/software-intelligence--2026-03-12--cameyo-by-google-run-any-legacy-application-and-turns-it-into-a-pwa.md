# Cameyo by Google: Run any legacy application and turns it into a PWA

## Summary
这不是一篇学术论文，而是 Cameyo by Google 的产品/方案介绍。它主张将传统遗留客户端应用通过虚拟化方式以网页应用/PWA 形式交付，从而支持更安全、跨设备、面向 Web 的工作环境。

## Problem
- 企业仍依赖大量遗留客户端应用，阻碍向 **web-first** 工作方式、ChromeOS 和现代协作套件迁移。
- 传统虚拟桌面方案“锚定过去”，在体验、运维和现代浏览器集成方面不够理想。
- IT 需要同时满足安全、跨设备访问、业务连续性与旧应用兼容，这对数字化转型很关键。

## Approach
- 用 **Virtual App Delivery** 取代传统虚拟桌面，把企业依赖的客户端应用以 **web apps/PWA** 的形式交付。
- 将应用访问能力集成到 **Chrome Enterprise** 安全企业浏览器中，使用户可“随时、任何设备”访问应用。
- 与 **Google Workspace** 协同，让组织在采用更协作的 web-first 生产力套件时，仍保留对关键客户端应用的访问。
- 与 **ChromeOS** 集成，在转向更安全、基于 Web 的操作系统时，通过浏览器无缝使用遗留应用。

## Results
- 文本**未提供可核验的模型、算法或基准实验定量结果**，因此没有论文式性能指标可报告。

- 还主张用户可通过浏览器从**anytime**、**any device** 访问所有应用，但未给出可量化的延迟、成本或生产力数据。
