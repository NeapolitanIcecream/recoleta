# Lynx Roadmap 2026

## Summary
## 摘要
这篇文章是 Lynx 在 2026 年的产品路线图，不是研究论文。它列出五个重点：改为月度发布、面向 AI 的文档和工具、更广的多平台支持包括桌面端、更接近生产可用的基础设施，以及更广泛的社区工作。

## 问题
- Lynx 是一个较新的跨平台框架，仍需要更稳定的发布节奏、更强的 API 稳定性，以及更清晰的升级指引，方便需要交付应用的团队。
- 开发者需要更完整的多平台覆盖、桌面端就绪能力、更丰富的原生能力，以及更好的调试和性能分析工具。
- 这份路线图也把 AI 辅助开发当作现实需求，因此框架需要能很好配合基于 LLM 的工具使用的文档、示例和 API。

## 方法
- 从双月发布周期改为从 2026 年年中开始的月度发布节奏，并改进发布说明、升级指南和版本可维护性。
- 通过让 API 更稳定、更有结构，并在 Lynx 网站上加入更适合 LLM 的文档、Agent Skills、工具和示例，提升对 AI 的适配。
- 继续推进 Android、iOS、Web、OpenHarmony 和桌面的多平台工作，依托开源的 Clay 渲染引擎；并通过 Lynxtron 加强 Lynx + Electron 的桌面支持。
- 通过 Sparkling 提供应用脚手架和原生导航，增加更多原生设备 API、更多 lynx-ui 组件、更强的动画和 CSS 支持、主线程脚本、运行时性能工作，以及更好的 DevTool、性能分析和诊断能力，从而补齐生产基础设施。
- 围绕 Rspack、React、Vue.js、Motion，以及可用于生产的 MiniApps 支持，扩大社区和生态工作。

## 结果
- 文章提到过去一年从 **v3.2 到 v3.6** 持续发布，**v3.7** 将结束当前的双月发布周期。
- 它称当前 Lynx 版本已经 **完整支持 Web 和 OpenHarmony**。
- 它说明 Lynx 现在正式支持 **Android、iOS、Web 和 OpenHarmony** 平台。
- 它说明开源的 **Clay** 渲染引擎面向 **macOS 和 Windows** 等桌面平台。
- 它列出了具体的生态输出，例如 **miso-lynx**、**Vue Lynx**、**Sparkling**、**lynx-ui**、**Lynx DevTool**，以及计划中的 **Lynxtron** 集成。
- 这段内容**没有提供基准测试数字、准确率指标、延迟数据，或与 Flutter、React Native、Kotlin Multiplatform、Compose Multiplatform 的一对一对比**。
