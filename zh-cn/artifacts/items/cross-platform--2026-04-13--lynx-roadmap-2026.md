# Lynx Roadmap 2026

## Summary
## 摘要
这篇文章是 Lynx 在 2026 年的产品路线图，不是研究论文。它提出了五个重点：按月发布、面向 AI 的文档和工具、更广的多平台支持并覆盖桌面端、更适合生产环境的基础设施，以及更广泛的社区建设。

## 问题
- Lynx 是一个较新的跨平台框架，对于需要持续发布应用的团队来说，仍然需要更稳定的发布节奏、更强的 API 稳定性，以及更好的升级指引。
- 开发者希望获得更完整的多平台覆盖、桌面端可用性、更丰富的原生能力，以及更好的调试和性能分析工具。
- 这份路线图也把 AI 辅助开发当作现实需求，因此框架需要能与基于 LLM 的工具良好配合的文档、示例和 API。

## 方法
- 从双月发布周期改为自 2026 年年中开始的按月发布，并改进发布说明、升级指南和版本可维护性。
- 通过让 API 更稳定、结构更清晰，并在 Lynx 网站上增加对 LLM 友好的文档、Agent Skills、工具和示例，提升对 AI 的适配。
- 继续推进 Android、iOS、Web、OpenHarmony 和桌面端的多平台支持；桌面端通过开源的 Clay 渲染引擎实现，并通过 Lynxtron 加深对 Lynx + Electron 的支持。
- 通过 Sparkling 提供应用脚手架和原生导航，增加更多原生设备 API、更多 lynx-ui 组件、更强的动画和 CSS 支持、Main Thread Scripting、运行时性能优化，以及更好的 DevTool、性能分析和诊断能力，补齐生产环境基础设施。
- 围绕 Rspack、React、Vue.js、Motion 以及面向生产环境的 MiniApps 支持，扩展社区和生态工作。

## 结果
- 文章称过去一年已经发布 **v3.2 到 v3.6**，并且 **v3.7** 将结束当前的双月发布周期。
- 它表示 Lynx 当前版本已经 **完整支持 Web 和 OpenHarmony**。
- 它表示 Lynx 现在对 **Android、iOS、Web 和 OpenHarmony** 提供官方平台支持。
- 它表示开源渲染引擎 **Clay** 面向的桌面平台包括 **macOS 和 Windows**。
- 它列出了具体的生态产出，例如 **miso-lynx**、**Vue Lynx**、**Sparkling**、**lynx-ui**、**Lynx DevTool**，以及计划中的 **Lynxtron** 集成。
- 摘录中**没有**给出与 Flutter、React Native、Kotlin Multiplatform 或 Compose Multiplatform 的基准测试数据、准确率指标、延迟数据或正面对比。
