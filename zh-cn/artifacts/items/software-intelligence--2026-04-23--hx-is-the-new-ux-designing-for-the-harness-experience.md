# HX Is the New UX: Designing for the Harness Experience

## Summary
## 摘要
这篇文章认为，软件设计需要从传统的用户体验转向面向 AI 代理的“Harness Experience”。它的核心观点是，当代理代表人行动时，设计重点应放在引导、信任、审计和交接上，而不是逐屏导航。

## 问题
- 传统 UX 假设人类通过屏幕、表单和漏斗流程来完成任务。
- 代理型软件可以通过调用 API、检查上下文并直接执行操作来绕过这些界面，这使许多旧的 UI 模式不再那么有用。
- 这很重要，因为人仍然需要控制代理的行为：他们必须设定意图、判断结果、检查决策，并在需要时介入。

## 方法
- 文章提出 **HX (Harness Experience)**，把它作为一种设计学科，用来处理人类与一个或多个执行型代理之间的关系。
- 它的核心机制很直接：把人看作代理工作的指挥者，而不是逐次点击操作产品的人。
- 文中提出三个设计支柱：**可引导性**（表达目标、约束和修正）、**透明性与可审计性**（用非工程师也能检查的形式说明代理为何这样行动），以及 **干预点**（让人类可以暂停、接管，并在不丢失上下文的情况下交还控制权）。
- 它还提出了代理系统的新设计原语：处理延迟、暴露不确定性，以及让验证更快，因为人的角色会更多转向质量控制。

## 结果
- 这段摘录**没有**提供实验、基准、数据集或定量比较。
- 它提出了一个结构性判断：在旅行预订这类任务中，代理可以跳过网站漏斗，通过 API 和用户上下文（如日历和偏好历史）来完成工作。
- 文中给了一个关于置信度的具体例子，称一个以 **78% confidence** 运行的代理应该暴露这种不确定性，而不是隐藏它。
- 它最强的主张是概念性的：为顺畅服从而构建的产品，一旦代理偏离用户意图就可能失效，而 HX 试图通过可引导性、可审计性和干预设计来降低这种风险。
