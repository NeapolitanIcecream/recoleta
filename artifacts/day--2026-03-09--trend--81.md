# 研究趋势日报：智能体从“会回答”走向“会工作”

## Overview
这一天的样本很小，但主题很集中。三条内容都指向同一个变化：AI系统正在从“会回答”走向“会工作”，而真正的分水岭不在模型本身，而在环境、流程和验证。

第一，智能体基础设施更强调持久环境。Terminal Use 把沙箱、文件系统、任务状态和部署流程打包成统一平台，核心卖点不是再造一个聊天接口，而是让工作区可以跨任务保存、共享和迁移。这很适合代码、研究和文档类智能体，因为这类任务天然依赖文件，而不是单轮对话。这里的信号是，智能体平台竞争开始落到“运行时能力”上。

第二，企业采用代理式AI时，越来越重视治理。测试自动化论文没有把AI描述成全自动替代者，而是把它放进一个有人审查的生产流程里。RAG（检索增强生成）先利用历史案例提高生成质量，再由执行、评估、报告和追踪模块形成闭环。更重要的是，作者给出了真实工业约束：即使规格清晰，AI也会因为缺少隐性知识而误解语义。所以趋势不是“少人化”，而是“人机协作流程化”。

第三，科研里的代理系统开始展现另一种价值。数学发现案例表明，LLM代理并不只是加速已有证明，也能帮助研究者发现异常、提出结构猜想、调用多种工具验证，再由 Lean 4 这样的形式化系统完成收尾。这种组合把探索、解释和验证拆开处理，降低了开放式研究中“想法很多但难以收敛”的问题。

如果把三条线索放在一起看，可以得到一个更高层的判断：下一阶段的AI产品和研究，重点会从“单模型能力展示”转向“任务环境 + 人机分工 + 结果验证”的系统设计。谁能把这三层接起来，谁更可能做出可持续的真实应用。

## Clusters

### 文件系统型智能体平台化

智能体基础设施开始从“聊天接口托管”转向“工作区托管”。代表案例将文件系统、沙箱、任务状态和消息流统一起来，并把文件系统从任务生命周期中解耦。这说明面向编码、研究和文档处理的智能体，正在更接近真实软件环境，而不是停留在无状态调用。

#### Representative papers
- [Launch HN: Terminal Use (YC W26) – Vercel for filesystem-based agents](https://news.ycombinator.com/item?id=47311657)


### 人机协作测试自动化进入治理阶段

在企业软件测试场景中，代理式AI更像“静默队友”，而不是完全替代者。重点不只是生成脚本，而是把检索增强生成、多智能体评估、执行日志和人工复核连成闭环。趋势信号很明确：落地重点从模型能力本身，转向可追踪、可审计、可交接的协作流程。

#### Representative papers
- [Human-AI Collaboration for Scaling Agile Regression Testing: An Agentic-AI Teammate from Manual to Automated Testing](http://arxiv.org/abs/2603.08190v1) — Moustapha El Outmani; Manthan Venkataramana Shenoy; Ahmad Hatahet; Andreas Rausch; Tim Niklas Kniep; Thomas Raddatz; …


### 神经符号科研从搜索走向结构发现

神经符号方法正在从“解题”走向“发现问题结构”。该案例里，LLM代理负责调用符号工具、搜索程序和优化程序，人类负责关键研究转向，最终再用形式化证明系统收敛结果。它展示了一个更务实的科研工作流：先用代理探索，再用数学结构解释，最后用形式化验证固化。

#### Representative papers
- [Agentic Neurosymbolic Collaboration for Mathematical Discovery: A Case Study in Combinatorial Design](http://arxiv.org/abs/2603.08322v1) — Hai Xia; Carla P. Gomes; Bart Selman; Stefan Szeider
