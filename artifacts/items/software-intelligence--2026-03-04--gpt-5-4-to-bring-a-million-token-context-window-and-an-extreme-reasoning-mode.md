# GPT-5.4 to bring a million-token context window and an extreme reasoning mode

## Summary
这是一则关于 OpenAI GPT-5.4 的媒体爆料，而非正式论文或技术报告。核心看点是上下文窗口可能扩展到 100 万 token，并加入面向高难任务的“极限推理”模式。

## Problem
- 现有模型在超长上下文任务中仍容易出错，尤其是需要持续数小时的复杂任务，如编程代理与长链条推理。
- 对研究者和高级用户来说，标准推理预算可能不足以处理最困难的问题。
- OpenAI 需要在与 Google、Anthropic 的竞争中缩小或消除上下文窗口能力差距。

## Approach
- 报道称 GPT-5.4 将上下文窗口提升到 **1,000,000 tokens**，以支持更长输入、更长任务轨迹和更多历史信息保留。
- 引入一种 **“extreme” thinking mode**，即在难题上分配更多计算资源，以换取更强推理能力而非更快响应。
- 目标之一是提升长任务中的可靠性，减少在多小时执行过程中的错误累计。
- 该能力被认为尤其有利于像 **Codex** 这样的编程代理，因为它们常需跨文件、跨步骤、长时间运行。

## Results
- 报道声称 GPT-5.4 的上下文窗口将达到 **1,000,000 tokens**，相较当前 GPT-5.2 的 **400,000 tokens** 超过 **2 倍**（约 **2.5×**）。
- 文中称这一规格将使 OpenAI 在上下文长度上与 **Google** 和 **Anthropic** 处于同一水平，但**没有提供基准测试、数据集或精确对比数值**。
- 报道进一步宣称新模型在“可运行数小时的长任务”上会**更可靠、错误更少**，但**没有给出定量指标**。
- “extreme” 推理模式被描述为面向研究者的高算力选项，但**没有公开其延迟、成本或准确率提升数据**。
- 因此，当前最强的具体结论仅是：**100 万 token 上下文**与**更高计算预算的推理模式**是被报道的主要突破点，而非已被官方验证的实验结果。
