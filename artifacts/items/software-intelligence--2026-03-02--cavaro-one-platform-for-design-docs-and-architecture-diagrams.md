# Cavaro – One platform for design docs and architecture diagrams

## Summary
Cavaro 是一个将设计文档、架构图和团队知识库整合到同一平台的文档协作产品。其核心卖点是把文档编写、图表绘制、实时协作与知识组织放在统一工作空间中。

## Problem
- 团队通常需要在文档工具和绘图工具之间切换，导致设计说明、架构图和决策记录分散，难以维护一致性。
- 技术文档、架构设计和团队知识缺乏统一管理，会降低协作效率，并增加知识丢失与版本混乱的风险。
- 对现代软件团队而言，能够协同维护官方文档和架构图很重要，因为这些内容直接影响设计评审、系统沟通和知识传承。

## Approach
- 提供统一平台，将富文本/Markdown 文档、嵌入式图表和团队知识库整合在同一工作区中。
- 内置拖拽式图表编辑器，可直接在文档里创建架构图、流程图、UML、ERD、BPMN、思维导图等。
- 支持实时协作机制，包括多人编辑、实时光标、评论、自动保存和完整版本历史。
- 通过工作区、权限控制、文档链接与层级结构，把分散文档组织成可浏览的知识网络。
- 提供云厂商官方图标库（AWS/Azure/GCP）以及 AI diagram generator、text-to-diagram、code-to-diagram 等能力以加速绘图。

## Results
- 文本未提供任何基准测试、实验数据或定量评测结果，因此没有可验证的性能数字、数据集或基线对比。
- 明确的产品能力声明包括：支持 **1000+** 官方 AWS、Azure 和 GCP 图标用于架构图绘制。
- 支持导出为 **PDF、PNG、SVG** 三种格式，并支持实时协作中的 live cursors 和 comments。
- 支持的图表类型明确包括 UML、flowcharts、mind maps、entity relationship diagrams、BPMN，以及架构图。
- 主要突破性主张不是算法性能，而是产品集成：相比需要分离文档和绘图应用的工具，Cavaro 声称通过“文档+图表”一体化平台减少上下文切换并改善协作。
- 还声称具备 AI diagram generator、text-to-diagram 和 create diagrams from code 能力，但 excerpt 中没有说明其模型、方法细节或效果对比。
