# The Effect of Code Obfuscation on Human Program Comprehension

## Summary
该论文研究代码混淆如何影响人类理解程序，核心关注输出预测任务中的正确率与作答时间。结论是：混淆通常会让人更慢、也更容易出错，但“混淆越强越难”并不总是单调成立，而且 Python 与 JavaScript 的表现不同。

## Problem
- 论文要解决的问题是：**代码混淆到底怎样影响人类程序理解**，而不仅是影响自动化指标；这很重要，因为混淆的真实目标本来就是增加人类逆向和理解难度。
- 现有常见假设是“混淆越强，人越难理解”，但作者指出这对人类未必总成立；不同混淆方式（重命名、误导性命名、控制流改写）可能作用机制不同。
- 还要回答：这种影响是否因语言而异，以及编程经验是否能跨语言迁移到混淆代码理解上。

## Approach
- 作者设计了一个受控人类实验，让参与者对给定 Python/JavaScript 函数和输入做**输出预测**，记录是否答对、耗时、以及自报语言经验。
- 构造 5 个混淆层级：L0 原始代码，L1 无信息重命名，L1b 具有语义但误导的对抗性重命名，L2 控制流改写/扁平化，L3 重命名+控制流组合。
- 数据来自 HumanEval-X 的 Python 和 JavaScript 子集，每种语言各抽取 10 段短小但非平凡的函数；共 50 名本科 CS 学生参加，每人完成 12 题，总计 600 次回答。
- 方法上的核心机制可以用最简单的话说：**通过逐步增加代码表面的“误导”和结构上的“绕路”，看人是否从快速直觉式阅读被迫切换到慢速、逐步跟踪式推理。**
- 作者用双系统理论解释现象：未混淆代码更容易依赖快速启发式（System 1），混淆代码尤其是控制流混淆会迫使人进入更慢、更费力的 System 2 推理。

## Results
- 基线 L0 准确率最高，为 **40.46% (53/131)**；各层级分别为 **L1: 38.68% (41/106)**、**L1b: 38.02% (46/121)**、**L2: 34.15% (42/123)**、**L3: 31.09% (37/119)**。相对 L0 的下降分别是 **-1.78%**、**-2.44%**、**-6.31%**、**-9.37%**。
- 组合混淆 **L3** 最难，说明重命名与控制流改写叠加会显著增加理解负担；作者还指出从 **L1b 到 L2** 的准确率跌幅最大，为 **3.87 个百分点**，表明控制流混淆比单纯误导性命名更伤害理解。
- 全体总体表现为 **36.5% 正确（600 次回答）**，说明任务本身就不简单；因此从 **40.46%** 下降到 **31.09%** 代表了显著、实际的人类理解损失。
- 作答时间上，L0 平均约 **140 秒**；引入混淆后 **L1–L3 平均耗时约增加 1 分钟**，支持“混淆把人从快速启发式推向慢速审慎推理”的说法。
- 在快/慢作答分组中，**L2** 的慢速组比快速组准确率高 **15.1 个百分点**，显示控制流混淆特别容易让快速直觉失效，但投入更多时间仍可能挽回一部分正确率。
- 正确答案的中位耗时为 **142.7 秒**，错误答案为 **127 秒**；但作者强调关系并非线性：适度变慢通常更好，**极长时间**往往意味着困惑而不是更高准确率。
