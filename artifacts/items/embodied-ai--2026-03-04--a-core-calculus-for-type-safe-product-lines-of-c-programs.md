# A Core Calculus for Type-safe Product Lines of C Programs

## Summary
本文提出了两个针对 C 语言软件产品线的形式化核心演算：LC 与带预处理器变体标注的 CLC，并给出一个族级类型系统来一次性保证所有变体都类型安全。其意义在于为含 `#define/#if` 的 C 产品线提供比逐变体检查更可行、也更正式的类型安全基础。

## Problem
- 目标问题：如何对使用 C 预处理器实现的软件产品线进行**族级**类型检查，从而保证所有可生成变体都是良类型 C 程序。
- 重要性：现实中的 C 软件产品线可能具有指数级配置空间，论文举例 Gentoo 在一个统计版本中有 **671617** 个特性、**36197** 个特性模型，理论上可达 **2^671617** 个产品；逐一生成并检查显然不可行。
- 空白点：已有工具可处理带预处理器的 C，但作者声称此前缺少一个“**C 的真子集**”上的、面向注释式 SPL 的**正式族级类型检查**核心演算。

## Approach
- 提出 **Lightweight C (LC)**：一个形式化的 C 核心子语言，覆盖结构体、指针、函数、条件表达式、表达式序列、运算符以及堆上的动态分配/释放等关键机制。
- 提出 **Colored LC (CLC)**：在 LC 上加入类似 ANSI C 预处理器的变体标注机制，使结构体定义、成员、函数、形参、实参和表达式序列元素都可按特性公式有条件地出现。
- 设计 **family-based** 类型系统：不是逐个生成变体再检查，而是把代码库、特性模型与注释知识一起分析，直接证明所有合法产品导出的程序都满足 LC 类型规则。
- 给出与实际 C 预处理习惯对应的编程模式，例如通过对表达式及逗号进行条件标注，保证删除部分片段后剩余代码仍然语法良构。
- 论文还说明 LC 程序与 ANSI/C24 类型系统的关系，并以带 **3** 个特性、**6** 个产品的队列 SPL 作为贯穿示例。

## Results
- 核心理论结果：作者明确声称其 CLC 类型系统**保证**“所有由 C 预处理器生成的程序都是良类型 C 程序”；文中还说明在第 6 节证明：所有良类型 CLC SPL 的所有变体都是良类型 LC 程序。
- 形式化产出包括 **2** 个核心演算：**LC** 与 **CLC**，以及 **1** 个族级类型系统；这是论文的主要技术贡献。
- 示例结果：队列产品线包含 **3** 个特性（`LAST`, `OCCURS`, `INTERVAL`）与 **6** 个合法产品/变体，文中指出这 **6** 个变体都是良类型 LC 程序。
- 规模动机数字：论文用 Gentoo 示例强调逐变体检查不可扩展，给出 **671617** 个特性、**36197** 个特性模型、最多 **2^671617** 个产品的数量级。
- 未提供实验型定量结果：摘录中**没有**出现运行时间、准确率、错误检出率或与基线工具的实证对比数字；最强结论是一个**形式化的类型安全保证**而非经验性能提升。
