# Creative Coding

## Summary
创意编码不是一篇研究论文，而是对一种以“表达性创作”而非“功能性软件”为目标的编程实践的概述。它说明了该领域的定义、历史、应用场景以及常用工具与硬件。

## Problem
- 解决的问题是：如何把编程用作艺术与表达媒介，而不仅仅是构建实用软件。
- 这很重要，因为它扩展了代码的用途，使程序可用于视觉艺术、声音艺术、游戏、装置、现场表演与原型设计等创作活动。
- 文中还指出一个更深层问题：在当代计算机艺术中，代码本身常被界面与用户体验讨论所掩盖，导致其作为创作材料的价值被低估。

## Approach
- 核心机制很简单：程序员/艺术家通过写代码来生成、控制或交互式地创作视觉、声音和装置作品，目标是“表达”而不是“功能”。
- 这种实践通常依赖专门面向快速原型与创意开发的编程库、框架和工具包，以降低制作艺术作品的门槛。
- 创意编码可结合硬件输入输出，例如麦克风、摄像头、深度相机、动作控制器、微控制器、MIDI 控制器、投影仪、LED 灯带、打印机和绘图仪。
- 文本从历史上将其放入 1960 年代计算机艺术、1980 年代 demoscene，以及后续关于“代码在艺术中地位”的理论讨论中。

## Results
- 文本**没有提供定量实验结果**、基准数据集、性能指标或与基线方法的数值比较。
- 最强的具体主张是定义层面的：创意编码被用于 live visuals、VJing、视觉艺术与设计、娱乐（如 video games）、艺术装置、projection mapping、sound art、advertising 和 product prototypes 等多个场景。
- 历史性主张包括：该实践起始于 **1960s**；**Compos 68** 在后续年代将编程用于艺术并进入国际展览；自 **1980s** 起 demoscene 程序员通过“demos”展示高技术视觉作品。
- 理论性主张包括：部分学者认为代码在计算机艺术中的“意向性与材料性”应被更认真地看待，而不应只强调 HCI、界面或用户。
