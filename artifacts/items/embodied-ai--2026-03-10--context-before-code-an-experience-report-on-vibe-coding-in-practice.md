# Context Before Code: An Experience Report on Vibe Coding in Practice

## Summary
这是一篇关于在真实可部署系统中使用“vibe coding”进行开发的经验报告。论文的核心结论是：AI 生成代码很擅长加速脚手架和常规实现，但在多租户隔离、权限控制、异步处理等生产级约束上仍高度依赖人工架构设计与审计。

## Problem
- 论文要解决的问题是：**对话式代码生成在生产环境约束下到底能做什么、不能做什么**，尤其是在多租户隔离、访问控制、检索范围限制和异步任务等关键架构要求下。
- 这很重要，因为现有研究更多关注生产力和主观体验，而**较少分析 AI 辅助编程在可部署系统中的可靠性与架构适配性**。
- 如果这些约束没有被正确落实，系统可能出现**跨项目数据泄漏、权限错误、阻塞式处理和技术债**，直接影响生产可用性。

## Approach
- 作者基于两个真实案例做回顾性经验研究：一个是**多项目代理学习平台**，另一个是**学术文档 RAG 系统**。
- 开发流程采用“**context before code**”：先明确功能需求和非功能架构约束，再用大模型生成 API、数据模型、工具函数和前端组件。
- 简单说，核心机制是：**让 AI 先写局部功能代码，人类负责定义边界条件、修正架构错误、补上系统级约束**。
- 作者通过**手工测试、代码检查、运行日志和后台任务监控**来验证租户隔离、角色权限、检索范围、引用对齐和异步执行是否真正成立。
- 论文进一步总结出一类“**non-delegation zones**”，即当前对话式代码生成难以可靠代理的架构任务，如隔离策略、权限策略、基础设施编排和验证逻辑。

## Results
- 论文**没有提供严格的定量实验指标**，也没有报告生产力提升百分比、缺陷率下降或与基线工具的数值对比。
- 在**2 个可部署系统**中，作者观察到 vibe coding 明显加速了**脚手架、CRUD、基础路由、序列化工具和 UI 模板**的开发。
- 但在两个案例中，AI 生成代码都反复遗漏关键约束：例如学习平台的早期路由**缺少 project-level filtering**，存在跨项目访问风险；内存更新流程最初也被实现为**同步执行**，违背异步要求。
- 在 RAG 系统中，早期生成版本会**输出引用但未验证答案与检索片段是否对齐**；文档嵌入初版也采用**同步处理**，导致大文件上传时响应时间过长。
- 作者最强的实证结论是：工程工作并未消失，而是从**样板代码编写**转移到**架构设计、隔离审计、策略制定、验证与监控**；文中表 1 明确列出了这种任务重分配。
- 论文声称的主要突破不是新的算法或 benchmark SOTA，而是提出了一个清晰实践结论：**在生产约束下，AI 辅助开发最适合作为“在明确架构边界内加速实现”的工具，而不是替代系统级设计。**
