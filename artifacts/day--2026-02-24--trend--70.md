# 研究趋势日报：机器人策略的“执行长度自适应”与可变形物体的硬件在环快速学习

> [!summary] TL;DR
> - “执行多少步”开始被当作可学习/可推断的决策变量，而不是固定超参数：AutoHorizon 用 VLA 内部注意力信号在测试时逐chunk选执行长度。（[VLA Knows Its Limits](http://arxiv.org/abs/2602.21445v1)）
> - 可变形物体学习更强调“选对误差”而不是“看更多数据”：关键时刻目标把优化压力集中在决定成败的瞬间。（[Learning Deformable Object Manipulation Using Task-Level Ite…](http://arxiv.org/abs/2602.21302v1)）
> - 两篇都在追求低成本适配：前者不改模型不再训练，后者一次示范 + 少量硬件试次，并展示跨条件迁移的可行性。（[VLA Knows Its Limits](http://arxiv.org/abs/2602.21445v1)；[Learning Deformable Object Manipulation Using Task-Level Ite…](http://arxiv.org/abs/2602.21302v1)）

## Overview
本日语料只有两篇论文，但共同指向一个清晰方向：**机器人系统正在从“离线学到一个固定策略”转向“在线判断自己何时该更保守/更激进”**。一篇工作把这种“在线判断”做在视觉-语言-动作模型（VLA，Vision-Language-Action）的**动作分块执行长度**上；另一篇工作把“在线修正”做在绳索这类可变形物体的**任务级迭代学习控制**上，并强调少数据、少试次。

## 这一天最值得记住的两句话
- VLA 的“动作chunk后半段”往往对新感知不够敏感，因此**固定执行长度会出现先升后降的性能曲线**；可以用模型内部注意力作为“可靠预测范围”的信号，在测试时自动选长度。（[VLA Knows Its Limits](http://arxiv.org/abs/2602.21445v1)）
- 面对绳索这类高自由度系统，用“整段轨迹误差等权”会把学习带偏；改为只在**关键时刻（critical point）**对齐物体状态，可显著提升少试次学习稳定性。（[Learning Deformable Object Manipulation Using Task-Level Ite…](http://arxiv.org/abs/2602.21302v1)）

## Clusters

### VLA 的动作分块：从固定执行长度到测试时自适应（AutoHorizon）

趋势点：VLA（Vision-Language-Action）在机器人操作里常用“动作分块（action chunking）”。模型一次预测一段动作，但系统只执行前缀（执行视野/执行长度）。这篇工作指出：**执行长度不是一个可忽略的工程细节**，它会导致成功率出现明显的“先升后降”。论文进一步通过分析 flow-based VLA 的注意力结构，提出在不改模型、无需再训练的前提下，用动作自注意力来估计模型的“可预测极限”，并据此在测试时为每个chunk动态选择应执行多少步，从而在“平滑性”和“反应性”之间自动折中。（[VLA Knows Its Limits](http://arxiv.org/abs/2602.21445v1)；摘要也概述了两种注意力现象与方法动机：[VLA Knows Its Limits](http://arxiv.org/abs/2602.21445v1)）

#### Representative papers

### 可变形物体的动态操作：任务级 ILC + 关键时刻目标

趋势点：绳索/布料等可变形物体操作很难建模，且纯数据驱动往往要大量示范或仿真。本日这篇工作把重点放在“**学得快**”与“**直接在硬件上学**”。方法是任务级 ILC（Iterative Learning Control，迭代学习控制）：优化对象不再是机器人自身的轨迹误差，而是被操纵物体（绳索）状态的任务误差；并提出关键时刻（critical point）目标，只在少数关键时间点对齐绳索状态，避免等权误差导致的发散。论文在 flying knot（飞结）任务上报告：一次人类示范起步，随后少量试次内达到高成功率，并能在多种绳索间用少量试次迁移。（[Learning Deformable Object Manipulation Using Task-Level Ite…](http://arxiv.org/abs/2602.21302v1)；摘要给出跨 7 种绳索与“10 次内 100%”等结果：[Learning Deformable Object Manipulation Using Task-Level Ite…](http://arxiv.org/abs/2602.21302v1)）
