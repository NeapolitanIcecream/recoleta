# Code Fingerprints: Disentangled Attribution of LLM-Generated Code

## Summary
This paper studies **attributing a piece of LLM-generated code to the specific model that produced it**, rather than merely judging whether it was “written by AI.” The authors propose DCAN, which performs multi-model, multilingual code source attribution by separately representing **task semantics** and **model-style fingerprints**.

## Problem
- Existing work can mostly distinguish only **human-written code vs. machine-generated code**, but in vulnerability triage, incident investigation, and license audits, what is actually needed is identifying **which provider/which LLM** generated the code.
- This problem matters because code generated by different models may entail different **security responsibilities, compliance risks, and provenance requirements**; without being able to locate the source, software governance becomes very difficult.
- The challenge is that, for the same task, different LLMs often produce code that is **functionally similar and syntactically similar**, with model differences showing up only in subtler stylistic, structural, and token-level preferences.

## Approach
- The authors split code representations into two parts: **source-agnostic** (task-related semantics shared across models) and **source-specific** (style fingerprints related to the generating model).
- They first use **UniXcoder** to encode code into a base vector `h_base`; then an MLP learns a shared semantic representation `h_com`, and the model-specific representation is obtained via **`h_spec = h_base - h_com`**.
- To ensure that `h_com` truly represents “task commonality,” the authors impose a representation consistency constraint on **code generated by different models for the same task**, minimizing their cosine distance in `h_com`.
- Finally, only `h_spec` is used for multi-class attribution, jointly optimizing **classification loss + representation consistency loss**; the default weight in the paper is `λ = 0.2`.
- For systematic evaluation, the authors construct the first large-scale benchmark for LLM code source attribution: **91,804** samples covering **4 models** (DeepSeek, Claude, Qwen, ChatGPT), **4 languages** (Python, Java, C, Go), and **2 settings** (with comments / without comments).

## Results
- In terms of dataset scale, the authors release a new benchmark containing **91,804** valid samples; **45,902** are in the no-comment setting, and **45,902** are in the with-comment setting.
- For the split, the training set contains **84,508** samples from **2,641** tasks; the test set contains **7,296** samples from **228** held-out tasks.
- In terms of coverage, the evaluation spans **4 mainstream LLMs × 4 programming languages × 2 generation settings**; tasks come from **2,869** LeetCode-style algorithm problems covering **21** algorithm tags.
- Regarding the method’s claims, the paper explicitly states that DCAN achieves “reliable attribution performance” under **multi-model, multilingual, and multi-setting** conditions, and demonstrates that **model-level code provenance is feasible**.
- However, in the provided excerpt, **no specific accuracy / F1 values are given, nor are quantitative comparisons with baselines such as GPTSniffer or CodeGPTSensor shown**, so more detailed performance gains in percentage terms cannot be restated.
