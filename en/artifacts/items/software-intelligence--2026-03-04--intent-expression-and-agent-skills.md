# Intent Expression and Agent Skills

## Summary
This article proposes a “documentation-as-skills” workflow for agentic coding, aiming to externalize developers’ intent more completely into reusable, reviewable documents, thereby improving the quality of code and design documents generated by agents. The core argument is that in agentic programming, clearly expressing intent provides more leverage than writing code directly.

## Problem
- Once agents write the code, the bottleneck shifts from “writing the implementation” to “whether requirements and design intent can be expressed clearly.”
- Relying only on one-off plan mode or conversational prompting is insufficient to support long-lived software knowledge that spans tasks, iterations, and version control.
- LLM-generated documentation carries risks such as misunderstanding the original intent, fabricating references, and creating an “illusion of completion”; without structured checks, these errors propagate into implementation.

## Approach
- The article introduces 3 reusable Claude Code skill files that form a documentation pipeline: `/flesh-out` (expand raw notes into structured documents), `/review-steps` (phased review and verification), and `/strong-edit` (criticize first, then revise).
- It adopts a “docs as code” approach, bringing documentation into the workspace and configuration management alongside source code, as a persistent shared reference for both humans and agents rather than ephemeral chat history.
- Each skill is broken into numbered stages, with developer confirmation points at every stage; after the agent completes a stage, it pauses and waits for human approval to reduce semantic drift and hallucination spread.
- In terms of division of labor, humans provide novel problems, intent, and engineering judgment; agents handle high-repetition work such as structuring, searching for industry practices, language polishing, consistency checks, and link validation.
- Agent behavior is controlled through vocabulary and process constraints—for example, “review” is more conservative, “flesh-out” is more generative, and “strong-edit” is more skeptical—allowing the same model to enter different working modes.

## Results
- In the writing case described in this article, raw notes of **727 words** were expanded by `/flesh-out` into **2279 words**, about **3.1x**; after human editing removed about **12%** (**2279 → 2007**), `/review-steps` added citations and observations to reach **2257 words**, and `/strong-edit` ultimately converged to about **1930 words**.
- The author reports that in Stage 0 of `flesh-out` and Stage 0 of `strong-edit`, the agent’s initial grasp of the core intent was “about **70%** correct,” with the remainder clarified through follow-up questions; this is one of the few explicitly stated empirical accuracy figures in the article.
- In another CPU model design case, raw notes of **169 words** were expanded into a **1962-word** design document, about **11.6x**; the agent then searched the source code and found **21+** user-mode assembly files across **10** library modules and **3** applications, overturning the key assumption that “only the kernel directly uses the ISA” and directly reshaping the design structure.
- The article also gives a failure case: `/review-steps` once added a **fabricated URL**, and the error passed through both human editing and `/strong-edit` unnoticed, ultimately prompting the author to add Stage 6: “verify one by one whether links resolve and whether they support the claims made in the text.”
- The author also cites an external survey saying developer trust in AI accuracy is only **43%**, to illustrate why phased checkpoints and traceable information sources matter in agentic software production.
- This is not a strict academic evaluation paper: it provides no standard benchmark data, control experiments, or statistically significant results; the strongest evidence is that across multiple real workflow cases, structured skills significantly amplified documentation output, exposed incorrect assumptions, and improved the stability of agents following process.
