# RepoLaunch: Automating Build&Test Pipeline of Code Repositories on ANY Language and ANY Platform

## Summary
RepoLaunch proposes a multi-agent system that can automatically build code repositories, run tests, and extract test results, aiming to cover **any language, any platform**. It further extends this capability into an automated SWE dataset generation pipeline for evaluating and training coding agents.

## Problem
- Installing dependencies, compiling, determining test commands, and handling platform differences for code repositories usually require substantial manual trial and error, making it difficult to process real GitHub repositories at scale.
- Existing methods are often limited to **Python/Linux** or rely on hard-coded rules, and cannot robustly cover multiple languages, multiple operating systems, and messy repository structures.
- This matters because training and evaluating LLM coding agents requires large numbers of **executable sandboxes** and **parsable test results**, while manual preparation far exceeds human capacity.

## Approach
- It proposes a three-stage multi-agent workflow: **Preparation → Build → Release**. It first scans the repository and selects a base image, then attempts dependency installation / compilation / test execution, and finally organizes a minimal reconstruction command, test commands, and a test log parser.
- The Preparation Agent reads the file tree, configuration files, and language hints, selects an appropriate base OS image, and passes repository context to downstream agents.
- In the Build stage, the Setup Agent repeatedly executes shell commands and web searches in a container to fix dependency and compilation issues; the Verify Agent checks whether regression tests can actually run and whether most tests pass, reducing hallucinated "false success."
- In the Release stage, the Organize Agent compresses historical commands into **minimal reusable reconstruction commands**, while also generating **test commands** and **log parsing functions**; it prioritizes structured outputs such as JSON/XML.
- Based on the above capability, the authors further construct an automated SWE task generation pipeline: researchers only need to design the tasks, while the rest—such as building, testing, state extraction, and evaluation—is completed automatically by RepoLaunch.

## Results
- The paper claims that RepoLaunch covers **9 programming languages** and two platform types, **Linux and Windows**, with an overall repository build success rate of about **70%**.
- In automated dataset creation, Build success rates are: Python **906/1200 = 75.5%**, C/C++ **297/400 = 74.3%**, C# **269/350 = 76.9%**, Java **267/350 = 76.3%**, JS/TS **483/700 = 69.0%**, Go **211/350 = 60.3%**, Rust **259/350 = 74.0%**, and Windows overall **258/400 = 64.5%**.
- In the Release stage, retention rates on successfully built samples are: C/C++ **261/297 = 87.9%**, C# **206/269 = 76.6%**, Java **203/267 = 76.0%**, JS/TS **422/483 = 87.3%**, Go **182/211 = 86.3%**, Rust **216/259 = 83.4%**, and Windows **206/258 = 79.8%**.
- The final **SWE-bench-Live/MultiLang** contains **413 tasks and 234 repositories**, exceeding SWE-bench-Multilingual’s **300 tasks and 41 repositories**; after filtering, the Windows subset retains **507** issues, and **400** are randomly sampled for evaluation.
- On the MultiLang benchmark generated by RepoLaunch, the best overall Linux average success rate among existing agent/model combinations is about **28.4%**: for example, SWE-agent+Claude-4.5 **28.4%**, ClaudeCode+GPT-5.2 **28.4%**, and ClaudeCode+Claude-4.5 **28.4%**; the best single-language result is ClaudeCode+Claude-4.5 reaching **44.1%** on Go.
- On the Windows benchmark, Win-agent with Claude-4.5 reaches **30.0%**, GPT-5.2 and DeepSeek-V3.1 are both **20.0%**, and Gemini-3 is **16.0%**. The authors also note that the main failure modes include timeouts, regression test execution failures, and difficulties with dependency resolution on Windows.
