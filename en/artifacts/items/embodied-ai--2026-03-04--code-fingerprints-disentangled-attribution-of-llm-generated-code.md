# Code Fingerprints: Disentangled Attribution of LLM-Generated Code

## Summary
This paper studies how to attribute a piece of LLM-generated code to the specific model that produced it, rather than merely determining whether it was “written by AI.” The authors propose DCAN, which separates task semantics from model-style fingerprints to identify code provenance across models and languages.

## Problem
- Existing work mostly focuses on the binary classification of “human-written code vs AI-generated code,” but in vulnerability triage, incident investigation, and licensing audits, the more critical question is identifying **which specific LLM** generated the code.
- This problem is difficult because when different models face the same programming task, the functional semantics and solution structure are often very similar, making the truly useful model-style signals weak and easily overwhelmed by task semantics.
- Without model-level attribution, it is difficult to support real-world needs such as software governance, accountability tracking, and compliance auditing.

## Approach
- The paper proposes **DCAN (Disentangled Code Attribution Network)**: it first uses the pretrained code encoder **UniXcoder** to extract code embeddings, then splits the representation into two parts: shared task semantics and model-specific style fingerprints.
- The core mechanism is very simple: for code generated by different LLMs on the same task, it forces their “common representations” to be close to one another; then it uses **the original representation minus the common representation** to obtain a “private representation” that is more oriented toward model features.
- Training uses two objectives: one is multi-class source identification on the “private representation”; the other imposes a same-task consistency constraint on the “common representation” (cosine distance loss), making it resemble task semantics rather than source style.
- The authors also construct the first large-scale benchmark for LLM code source attribution: **91,804** samples covering **4 LLMs** (DeepSeek, Claude, Qwen, ChatGPT), **4 languages** (Python, Java, C, Go), and distinguishing between **with comments / without comments** settings.

## Results
- The abstract and provided excerpt explicitly claim that DCAN achieves “reliable attribution performance” under **multiple models, multiple languages, and multiple settings**, indicating that model-level code attribution is feasible.
- Regarding dataset scale, the authors provide specific figures: **91,804** total valid samples, of which **45,902** are without comments and **45,902** are with comments; **84,508** in the training set and **7,296** in the test set; **2,869** tasks in total, including **2,641** training tasks and **228** test tasks.
- The concrete scale of candidate source models and coverage is: **4 LLMs × 4 programming languages × 2 generation settings**, with about **11,476** samples per model per setting, supporting systematic comparison.
- However, in the excerpt currently provided, **no specific accuracy/F1 values appear, nor are there percentage-point gains over baselines such as GPTSniffer and CodeGPTSensor**, so it is not possible to report a quantitative SOTA margin.
- The strongest verifiable conclusion from the current text is that the authors propose a **passive** attribution framework that does not require modifying the generation process, and they release the **first public large-scale benchmark** together with the implementation code.
