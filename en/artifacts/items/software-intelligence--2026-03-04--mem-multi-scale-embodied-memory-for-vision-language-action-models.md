# MEM: Multi-Scale Embodied Memory for Vision Language Action Models

## Summary
MEM adds hierarchical, multimodal memory to robot vision-language-action models: it uses video to remember fine-grained details from the last few seconds, and language to remember semantic events from much earlier. It is designed to help robots maintain context over real long-horizon tasks lasting up to about 15 minutes, and to adjust in place after manipulation failures.

## Problem
- Traditional robot policies usually look only at the current observation, or directly stuff a short history into the model; for tasks lasting several minutes, this is infeasible in both compute and latency.
- Long-horizon robot tasks require **memory at different granularities**: short-term memory needs visual detail to handle occlusion and failed grasps/regrasping, while long-term memory needs semantic summaries to remember which steps have already been completed.
- A single memory form struggles to satisfy both needs, which limits long-horizon manipulation, task progress tracking, and robust recovery in real-world settings.

## Approach
- Proposes **MEM (Multi-Scale Embodied Memory)**: the policy is split into a high level and a low level. The high level uses the current observation, task goal, and existing language memory to generate the next subtask instruction and update the language memory; the low level uses a recent observation history and the subtask to output actions.
- **Long-term memory** is represented with natural-language summaries: the model maintains a continuously updated semantic memory that keeps only information still relevant for future execution, while compressing or discarding redundant details.
- Supervision data for language memory is generated by an external LLM: based on subtask sequences and success/failure markers, it produces summaries of “which past events still matter,” used to train the high-level policy to learn when and how to update memory.
- **Short-term memory** is represented by an efficient video encoder: causal temporal attention is inserted into the ViT, compressing multi-frame visual history into the current-timestep representation before sending it to the VLA backbone; complexity drops from naive spatiotemporal attention `O(n^2K^2)` to `O(Kn^2 + nK^2)`.
- This video encoder **adds no new learnable parameters** and can be initialized directly from a pretrained single-image VLM/ViT; in experiments, pretraining uses 6-frame inputs, while post-training/testing scales to **18 frames, 54 seconds** of observation memory, and combined with language memory covers tasks up to **15 minutes** long.

## Results
- The paper claims MEM enables policies to complete tasks requiring memory over as long as **15 minutes**, such as **kitchen cleanup** and making a **grilled cheese sandwich**, as well as multi-stage pick-place and cleanup in recipe setup scenarios.
- In recipe setup, the authors use **42 training recipes** and evaluate on **5 test recipes** with **unseen kitchens and unseen objects**; all evaluations are **10 rollouts per policy/task**, reporting mean ± standard error.
- Compared with memoryless **π0.6**, MEM is described as “significantly improving success rate” on long-horizon tasks; ablations also show that **both short-term video memory and long-term language memory are indispensable**. However, the provided excerpt **does not include the specific success-rate values from the figure**, so exact percentages cannot be transcribed item by item.
- Without video memory, the robot gets “stuck” on tasks like washing dishes and wiping counters because it lacks recent context; without language memory, it struggles to remember recipe steps, which doors/cabinets have been opened or closed, and which areas have already been cleaned.
- Compared with **naive language memory** (simply concatenating all past subtask instructions), MEM’s **compressed language summaries** are stronger; the authors attribute the advantage to reduced train-inference distribution shift. - On short-horizon tasks, MEM also claims **in-context adaptation**: for example, when abnormal table height causes chopstick grasping errors, or when the fridge door opening direction is unclear, it can adjust grasping/opening strategies based on recent failure history; however, this excerpt does not provide corresponding quantitative metrics.
