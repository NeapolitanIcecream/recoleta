Summary

This paper evaluates the ability of GPT-4, Claude 3, and Gemini to automatically generate Behavior-Driven Development (BDD) scenarios, and constructs a dataset containing 500 industrial-grade user stories, requirement descriptions, and corresponding BDD scenarios. The study shows that different models and prompting strategies each have their own optimal configurations, and that high-quality requirement descriptions are more important than user stories alone.

Problem

  • The paper aims to solve how to use large language models to automatically generate high-quality BDD/Gherkin scenarios, so as to reduce the time required to manually write test scenarios, lower the experience barrier, and address insufficient coverage.
  • This matters because manually writing BDD scenarios is both slow and unstable in agile and continuous delivery environments, easily misses edge cases, and becomes a bottleneck for testing and delivery.
  • Existing work lacks systematic evaluation of this task, especially comprehensive comparisons across models, prompting methods, input types, and evaluation dimensions.

Approach

  • A new dataset was constructed: 500 samples from 4 industrial software products, each containing a user story, requirement description, and a manually written reference BDD scenario.
  • Three LLMs were used to generate BDD scenarios: GPT-4o, Claude 3 Opus, and Gemini 1.5 Flash, and their performance on this task was compared.
  • A multidimensional evaluation framework was adopted: text similarity, semantic similarity, LLM-as-a-judge evaluation, and human expert evaluation, rather than only checking whether the syntax is correct.
  • Four factors were systematically examined: baseline generation performance, prompting strategy (zero-shot, few-shot, chain-of-thought), input form (user story only, requirement description only, both combined), and sampling parameters (temperature, top_p).
  • The core mechanism can be simply understood as feeding natural-language requirements to an LLM, having it directly generate BDD scenarios in Given/When/Then format, and then judging generation quality using automatic metrics and human evaluation.

Results

  • The dataset contains 500 samples from 4 real industrial software products; the authors claim this is the first public BDD scenario dataset of its kind.
  • On text and semantic similarity metrics, GPT-4 scores higher; however, in human expert evaluation and LLM-based evaluation, scenarios generated by Claude 3 are rated best.
  • As an automatic evaluator, DeepSeek shows stronger correlation with human judgment than with text/semantic similarity metrics; the summary does not provide specific correlation coefficient values.
  • Prompting effectiveness shows clear model dependence: GPT-4 performs best with zero-shot, Claude 3 benefits most from chain-of-thought, and Gemini performs best under few-shot.
  • Input quality significantly affects results: detailed requirement descriptions alone can generate high-quality scenarios, while using only user stories yields lower-quality results; the summary does not provide specific score differences.
  • For sampling parameters, the optimal cross-model setting is temperature = 0 and top_p = 1.0; the authors state this configuration produces the highest-quality BDD scenarios, but the provided text does not include specific absolute metric values or statistical significance.
