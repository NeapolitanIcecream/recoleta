Summary

This is a collection of common interface design mistakes serving as a "negative example" resource, aiming to help developers avoid repeating the same errors by showcasing bad examples. It is not a technical paper, but rather a practice-oriented usability and interaction design critique resource.

Problem

  • The problem it addresses is that common design mistakes repeatedly appear in software interfaces and are difficult to identify and correct in time, leading to poor usability, user confusion, and degraded experience.
  • This matters because interface flaws directly affect users' efficiency in completing tasks, their satisfaction, and the overall quality of software products.
  • The resource also attempts to establish a public reference for "design patterns that should be phased out" and "design patterns worth emulating."

Approach

  • The core method is simple: collect and publicly showcase examples of poor interface design, organizing them into a "Hall of Shame" case repository.
  • By "pointing out what was done wrong," it helps developers build awareness of anti-patterns so they can proactively avoid similar issues in future designs.
  • At the same time, it is paired with the "Hall of Fame" concept, emphasizing that it not only criticizes bad design but also encourages learning from good design practices.
  • It uses an open submission mechanism, inviting readers to submit candidate cases and continuously expand the collection.

Results

  • The provided excerpt does not give any quantitative experimental results; there is no dataset, evaluation metric, baseline method, or numerical improvement.
  • The strongest specific claim is that highlighting common interface design mistakes can help developers avoid making similar mistakes.
  • The text also explicitly states that its goal is to continuously collect "design practices worthy of extinction" and "design practices worthy of emulation," forming a long-term reference resource.
  • Judging from its form, it is more like an experiential design case repository than a research work proposing a new algorithm, system, or reproducible experimental results.
