Authors

Summary

This paper proposes FP-Predictor, which uses a graph convolutional network and code property graphs to determine whether static security analysis reports are true positives or false positives, with the goal of reducing the burden on developers dealing with false alarms. It focuses on Java cryptographic API misuse scenarios and shows strong performance on two benchmarks, though part of the results depends on label interpretation after manual review.

Problem

  • Static Application Security Testing (SAST) tools generate large numbers of false positives, wasting developer time, slowing down the remediation of real vulnerabilities, and weakening trust in automated security analysis.
  • Rule- or pattern-matching approaches struggle to understand complex relationships in code such as syntax, control flow, and data dependencies, making it difficult to determine whether a report is actually exploitable.
  • The paper aims to solve the problem of automatically predicting whether a SAST report is a false alarm or a true finding, especially for Java cryptographic API misuse, which is important for improving the efficiency of security scanning workflows.

Approach

  • Each vulnerability warning reported by CogniCrypt is converted into a method-level Code Property Graph (CPG), combining the AST, CFG, and PDG into a unified graph representation.
  • Three types of node features are used to represent code: Word2Vec vectors of Jimple statements, one-hot encodings of node types, and a binary marker indicating whether a node is a violation node.
  • A GCN aggregates neighborhood information over the graph and outputs a �[0,1]� score representing the confidence that the warning is a false positive; the demonstration uses a 0.8 threshold, above which the warning is classified as a false positive.
  • The model is trained and tested on CamBenchCAP (431 labeled samples, 80/20 split), and then evaluated for generalization on a different benchmark, CryptoAPI-Bench.
  • The paper also performs manual security review of error cases, arguing that some “misclassifications” actually reflect the model making more conservative, security-oriented judgments.

Results

  • On the CamBenchCAP test set, the model reports 100% accuracy (80/20 train-test split); the authors also acknowledge that the dataset is relatively small (431 labeled cases) and structured, which may overestimate generalization ability.
  • On CryptoAPI-Bench (evaluating only intra-class scenarios and excluding 21 inter-class cases): for true-positive samples, the model correctly identifies 89/91 and misclassifies 2/91 as false positives, for about 97.8% accuracy.
  • For false-positive samples, using the original benchmark labels, it correctly identifies only 1/27, i.e. about 3.7%; this indicates that, judged directly against the original labels, the model is very weak at identifying false alarms.
  • However, after manual review, the authors argue that among the initial 26 “misclassifications,” 22 actually contain real security issues or at least poor cryptographic practices; accordingly, the “effective accuracy” on the false-positive subset rises from 1/27 (3.7%) to about 23/27 (85.2%).
  • Based on this manual relabeling, the paper claims overall accuracy can reach 96.6%; if 3 “debatable but still bad-practice” labels are not changed, the overall accuracy is 94.1%.
  • Training cost is low: about 5 minutes on Intel i7-11700K + RTX 4090 + 64GB RAM. The authors also note that the current CPG lacks interprocedural call relationships, which may limit performance on cases involving if/control-flow dependencies.
