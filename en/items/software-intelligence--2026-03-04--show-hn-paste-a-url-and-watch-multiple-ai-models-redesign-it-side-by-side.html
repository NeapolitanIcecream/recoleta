Summary

This is a multi-model parallel generation product for website redesign: users paste an existing site URL and a description of their goals, and the system has multiple AIs produce different redesign proposals simultaneously, with support for visual fine-tuning and code export. It emphasizes letting AI handle repetitive front-end build work while leaving design judgment and business goal definition to humans.

Problem

  • Website redesign usually requires repeatedly switching between design exploration, front-end implementation, and proposal comparison, which is time-consuming and depends on manually producing multiple alternative versions.
  • A single model often provides only one interpretation path, making it difficult to quickly compare different design directions, which affects business goals such as conversion, modernization, and usability.
  • When non-technical or front-end-light users want to turn design results into deployable code, they usually still need an additional engineering implementation step.

Approach

  • Users enter a website URL and natural-language goals, such as improving conversion, modernizing the interface, or improving usability, and the system starts generation based on that input.
  • Multiple AI models run in parallel and each independently generates redesign proposals; the core mechanism is “same input, multiple models, simultaneous outputs, side-by-side comparison.”
  • Users can view results from different models side by side and choose the more suitable direction, rather than relying on a one-off output from a single model.
  • The selected proposal can be adjusted in Shuffle by dragging and dropping components, fonts, and colors, with no need to hand-write CSS.
  • The final output can be exported as production-ready code; the formats listed in the text include Next.js, Laravel, WordPress, and HTML/CSS.

Results

  • The text does not provide standard academic experiments, benchmark data, or quantitative metrics, so there are no verifiable performance numbers to report.
  • Clear product-level capability claim: multiple models generate independent redesign proposals in “parallel” and support “side-by-side” comparison.
  • Clear delivery claim: export is available to 4 named formats/ecosystems — Next.js, Laravel, WordPress, HTML/CSS.
  • Clear interaction claim: supports no-code visual fine-tuning, including editing components, fonts, and colors.
  • Value proposition claim: by letting AI handle repetitive front-end build work, users can focus their effort on design judgment, goal setting, and business context.
