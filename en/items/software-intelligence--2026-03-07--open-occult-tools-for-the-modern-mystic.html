Summary

Open Occult is an open-source collection of resources and tools for occultism, spirituality, and divination practices, providing structured datasets, APIs, a Discord bot, and a learning platform. It is more like a community-driven software and content infrastructure project than a research paper proposing a new algorithm.

Problem

  • The project attempts to solve the problem that occultism, divination, and related cultural knowledge are scattered, hard to access, and lack a unified structured toolchain, which is inconvenient for learners, community operators, and developers alike.
  • It matters because if related knowledge, data, and interactive tools are standardized and made open, users can more easily learn, query, build applications, and engage in community interaction.
  • From a software perspective, it focuses on the lack of “accessible resources and development interfaces,” rather than code intelligence, software agents, or automated software production itself.

Approach

  • The core mechanism is simple: organize mythology, botanicals, runes, tarot, and similar content into open-source, JSON-formatted, category-organized datasets for direct use by both people and programs.
  • On top of the data, it provides Interactive APIs so developers can access curated data resources such as divination, correspondences, and spiritual tools.
  • It provides a Discord bot Cabot based on Node.js, Discord.js, and TypeScript, integrating queries, divination, and interactive commands into community settings.
  • Through documentation, API Reference, Data Structure Guide, contribution workflows, i18n support, and the upcoming Runeva learning portal, it builds a mechanism for community collaboration and continuous expansion.

Results

  • The text does not provide any quantitative experimental results; there is no dataset scale, user count, accuracy, latency, baseline comparison, or ablation results.
  • The strongest concrete claim is that the project already provides multiple categories of open-source data resources covering topics such as mythology, botanicals, runes & symbols, tarot & divination.
  • The project claims to provide Interactive APIs, Discord bot (Cabot), JSON-formatted datasets, i18n support, and to use the Node.js / Discord.js / TypeScript tech stack.
  • It also claims to support community collaboration through GitHub repositories, contribution workflows, documentation, a website, a Discord Server, and Discussions, but provides no adoption or effectiveness metrics.
  • Therefore, its “results” are more accurately described as a well-organized open-source platform and tooling ecosystem, rather than an empirically validated research breakthrough.
