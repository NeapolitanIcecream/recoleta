Authors

Summary

This paper proposes MIST-RL, which shifts unit test generation from “generate more tests” to “make every new test more useful.” It uses reinforcement learning to reward tests that can discover new bugs, thereby producing shorter but stronger test suites.

Problem

  • Problem addressed: Code generated by LLMs often contains hidden logic errors, so automatically generated unit tests are needed to verify and rerank candidate code; however, existing methods mainly rely on increasing the number of tests, which quickly leads to diminishing returns.
  • Why it matters: Redundant tests bring high computational cost, low gains in fault detection, and weaken the discriminative power of tests as verifiers, allowing incorrect code to still pass screening.
  • The paper’s core claim is: the key is not the number of tests, but whether each test can kill a previously unkilled mutant, i.e., whether it provides new verification information.

Approach

  • Test generation is modeled as a sequential decision process: the model generates tests one by one, and at each step refers to the function itself and the tests already generated.
  • Mutation testing is used to construct the reward: if a new test can kill a mutant that previously survived, it receives a reward; if it only repeatedly covers already discovered error patterns, it gets no additional gain.
  • An incremental mutation reward is designed: only mutants corresponding to “newly added kills” are rewarded, and they can be weighted by the complexity of the code region, encouraging the model to prioritize finding harder bugs.
  • A dynamic redundancy penalty is designed: as the generation step increases, if a new test adds no new utility, a larger penalty is applied to suppress test bloat.
  • The policy model is trained with GRPO, and a test quality term is added to prevent the model from gaming the reward by piling up low-quality assertions.

Results

  • On HumanEval+, MIST-RL achieves a Mutant Kill Rate = 74.03%, an improvement of +28.5 percentage points over CodeRM-8B’s 45.53%; it is also higher than the larger Qwen3-14B’s 58.69%. The average test suite length is 6.14 vs. 7.61, a 19.3% reduction relative to CodeRM-8B.
  • On MBPP+, MIST-RL achieves a Mutant Kill Rate = 70.27%, higher than CodeRM-8B’s 61.08% and Qwen3-14B’s 66.50%; the average length is 5.17 vs. 6.55, a 21.1% reduction relative to CodeRM-8B.
  • On DS-1000, MIST-RL achieves a Mutant Kill Rate = 57.90%, higher than CodeRM-8B’s 49.08% and Qwen3-14B’s 53.20%; the average length is 5.78 vs. 7.37, also more compact.
  • As a downstream verifier, in HumanEval+ reranking, when the number of candidates is N=10, Pass@1 = 48.78%, higher than CodeRM-8B’s 45.73% and Qwen3-14B’s 44.51%, and 3.05 percentage points above the SOTA baseline reported in the paper.
  • In N=20 reranking, MIST-RL still performs best: 62.80%, compared with CodeRM-8B’s 61.59% and Qwen3-14B’s 55.49%.
  • Ablation experiments show: after removing the incremental reward, the mutation score on HumanEval+ drops from 74.03% to 65.1%; after removing the dynamic penalty, the average length surges from 6.14 to 14.20, showing that the two components contribute to performance gains and redundancy reduction, respectively.
